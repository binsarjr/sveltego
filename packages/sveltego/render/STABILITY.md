# Stability — packages/sveltego/render

Last updated: 2026-04-30 · Version: pre-alpha

Tiers per [RFC #97](https://github.com/binsarjr/sveltego/issues/97).

## Tier legend

| Tier | Promise |
|---|---|
| `stable` | Won't break within the current major. Add-only changes; behavior changes documented in CHANGELOG. |
| `experimental` | May break in any minor release. Marked `// Experimental:` in godoc. |
| `deprecated` | Scheduled for removal. Marked `// Deprecated:` in godoc. Removed in next major. |

## Stable

- `render.Writer` — pooled SSR output buffer; the primary rendering primitive consumed by all generated page handlers
- `render.New` — allocates a fresh Writer (not pooled)
- `render.Acquire` — acquires a Writer from the pool
- `render.Release` — returns a Writer to the pool

## Experimental

- `render.Writer.WriteString` — appends a raw string without escaping
- `render.Writer.WriteRaw` — appends a raw string without escaping (alias intent differs; may consolidate)
- `render.Writer.WriteRawBytes` — appends raw bytes without escaping
- `render.Writer.WriteEscape` — appends a value with HTML text escaping
- `render.Writer.WriteEscapeAttr` — appends a value with HTML attribute escaping
- `render.Writer.WriteJSON` — serializes a value as JSON into the buffer
- `render.Writer.Bytes` — returns the accumulated buffer contents
- `render.Writer.Len` — returns the byte length of the buffer
- `render.Writer.Reset` — resets the buffer without releasing memory
- `render.Writer.FlushTo` — copies buffer to an io.Writer and flushes if the writer implements http.Flusher

## Breaking change procedure

Any `stable` symbol change requires an RFC issue describing the change and migration path, a PR that adds the new API alongside the old, at least one minor cycle with `// Deprecated:` godoc, and removal in the next major bump with a CHANGELOG entry. See [RFC #97](https://github.com/binsarjr/sveltego/issues/97) for the full procedure; CHANGELOG entries are generated by release-please.

## How to mark new symbols

Every exported symbol added in a PR **must** add a corresponding row to this file in the same PR. Place the row under `## Experimental` unless you have explicit approval to mark it `stable`. If the symbol is intentionally internal, place it under `## Internal-only`.
