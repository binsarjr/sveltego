# Stability — packages/sveltego/internal/codegen

Last updated: 2026-04-30 · Version: pre-alpha

Tiers per [RFC #97](https://github.com/binsarjr/sveltego/issues/97).

**Note:** This is an `internal/` package. Go's visibility rules already
prevent external import; this file documents the stability contract for
consumers within the same module (primarily `cmd/sveltego` and tests).
Only the public entry points listed below are tracked; all other unexported
and internal helpers are exempt and may change at any time.

## Tier legend

| Tier | Promise |
|---|---|
| `stable` | Won't break within the current major. Add-only changes; behavior changes documented in CHANGELOG. |
| `experimental` | May break in any minor release. Marked `// Experimental:` in godoc. |
| `deprecated` | Scheduled for removal. Marked `// Deprecated:` in godoc. Removed in next major. |

## Stable

(none yet — codegen API is still converging)

## Experimental

- `codegen.Options` — options for single-page Generate calls
- `codegen.LayoutOptions` — options for GenerateLayout calls
- `codegen.ErrorPageOptions` — options for GenerateErrorPage calls
- `codegen.Generate` — emits a Go source file for a `+page.svelte` fragment
- `codegen.GenerateLayout` — emits a Go source file for a `+layout.svelte` fragment
- `codegen.GenerateErrorPage` — emits a Go source file for a `+error.svelte` fragment
- `codegen.BuildOptions` — options for a full project code generation run
- `codegen.BuildResult` — result returned by Build (generated file list, diagnostics)
- `codegen.Build` — runs a full codegen pass over a project's routes directory
- `codegen.ManifestOptions` — options for GenerateManifest
- `codegen.GenerateManifest` — emits the route manifest Go source file
- `codegen.CodegenError` — structured error carrying source position information
- `codegen.Builder` — low-level line-oriented code builder used internally and in tests
- `codegen.EnvLookup` — function type for environment variable substitution

## Deprecated

(none)

## Internal-only (do not import even though exported)

All symbols in this package not listed above (e.g. `Builder.Line`, `Builder.Linef`, `Builder.Indent`, `Builder.Dedent`, `Builder.Bytes`, `Builder.Err`, `Builder.Fail`) are exported solely for use within this package's test suite and `cmd/sveltego`. They are not part of any stability promise and may change without notice.

## Breaking change procedure

Any `experimental` entry-point change that affects `cmd/sveltego` or generated output format must be described in a PR that updates the golden test corpus and CHANGELOG. See [RFC #97](https://github.com/binsarjr/sveltego/issues/97) for the full procedure; CHANGELOG entries are generated by release-please.

## How to mark new symbols

Every exported entry point added in a PR **must** add a corresponding row to this file in the same PR. If the symbol is only needed for intra-package tests, place it under `## Internal-only` with a note explaining its scope.
