# Lessons — sveltego

## 2026-04-29 — Initial R&D

### Insight

- SvelteKit's `Server.respond(Request) → Promise<Response>` is a small contract — Web standards plus optional `AsyncLocalStorage`.
- "Webcontainer mode" was the escape hatch we considered to avoid `AsyncLocalStorage`: serialize requests in runtimes without ALS. It works but caps throughput.
- goja is pure Go but not a drop-in modern JS runtime — partial ESM, no dynamic import, zero Web APIs.
- v8go is the perf king but cross-compile is painful (prebuilt V8 bindings per target).
- subprocess Bun is fastest path to production but is not "true embed" — you ship a 50MB+ runtime alongside the Go binary.

### Self-rules

1. Don't claim "embed" without distinguishing in-process runtime vs ship-binary. Ask the user.
2. Modern SvelteKit bundles use ESM + dynamic import. Runtimes lacking either need a transpile step in the adapter.
3. Web API polyfills in goja are scope creep. Estimate ~70% of total effort.
4. Avoid "production-ready" claims for early PoCs — tier probabilities (PoC vs full vs production).

## 2026-04-29 — Pivot to Go-native rewrite

### Insight

- All JS runtimes bond CPU to a JS engine. Even when the throughput is "OK" (Bun subprocess), the concurrency model is foreign to Go: no goroutines, no `context.Context`, IPC overhead per request.
- Adapters layered on top of SvelteKit-the-JS-server inherit every limitation of the chosen runtime. Going faster than the runtime is impossible.
- The SvelteKit *shape* (file convention, Load/Actions/hooks, layouts) is what users want — not the SvelteKit *implementation*.
- Codegen `.svelte` → Go source is feasible: Svelte 5 templates have a tractable subset, and the `<script>` block can host Go directly when we declare expressions are Go-native.
- Once expressions are Go, we can run `go/parser.ParseExpr` at codegen for validation — type errors surface at build, not runtime.

### Self-rules

1. When the user says "I want X performance," check whether the chosen runtime can ever reach it. If not, propose a different architecture before more polyfill work.
2. Performance ceilings are hard. The runtime defines the max throughput; nothing above it is recoverable via code.
3. Familiar shape (file convention, mental model) is the actual product. Don't conflate it with the upstream implementation.
4. Codegen beats runtime interpretation for SSR every time — static decisions cost nothing per request.

### Decisions captured

- Repo: `binsarjr/sveltego` (private at start).
- Build tool: pure Go. No Node/Bun runtime on the server. Vite stays at build time for the client bundle.
- Expressions: Go-native (PascalCase fields, `nil`, `len()`). No JS-to-Go translator.
- Target: Svelte 5 (runes) only. Skip Svelte 4 legacy syntax.
- Performance target: 20–40k rps for mid-complexity SSR.

## 2026-04-29 — Issue authoring standard

### Insight

- An issue list of ~70 items doesn't speak for itself. Bullet-only checklists without context burn future contributor time looking up "what does this mean."
- Industry-standard issue body is a contract: Summary, Background, Goals, Non-Goals, Detailed Design with code, Acceptance Criteria, Testing Strategy, Out of Scope, Risks & Open Questions, Dependencies (Blocks/Blocked by), References.
- Switching repo language to English mid-project is cheap if done in one pass.

### Self-rules

1. When seeding a roadmap, write each issue as if a stranger will pick it up — context plus contract.
2. Cross-reference dependencies explicitly (Blocks / Blocked by). Don't make readers reconstruct the order.
3. Ship code samples in design sections. Words drift; signatures don't.
4. One language per repo. If switching, batch the migration in a dedicated pass.

## 2026-04-29 — Foundation-first to prevent AI hallucination

### Insight

- An AI agent (or new contributor) joining mid-project hallucinates conventions when the conventions live nowhere central.
- Pre-alpha is the cheapest moment to encode every cross-cutting rule: code style, error handling, logging, ctx propagation, API stability tiers, release process, CI gates, golden testing, bench thresholds.
- Single source of truth per concern. AGENTS.md → auto-sync to .cursorrules + copilot-instructions. Hand-maintaining four copies guarantees drift.
- "Read in this order" list at the top of CLAUDE.md is the cheapest defense against hallucination. The list points at issues #95–105 even before those land as docs, because the issue body is itself the spec.
- A monorepo with N packages needs a per-package STABILITY.md, CHANGELOG.md, and optional CLAUDE.md. Centralized docs help discovery; package-local docs anchor scope-specific patterns.

### Self-rules

1. Encode conventions as foundation issues, not as folklore. If a rule isn't in a referenceable doc or issue, it doesn't exist.
2. AGENTS.md is the master; tool-specific files are generated from it. Never edit `.cursorrules` or `.github/copilot-instructions.md` by hand.
3. CLAUDE.md opens with a numbered "read in this order" list, including issue numbers. Future Claude instances read it before acting.
4. Any new convention adds a checklist item to the PR template Definition of Done. If the DoD doesn't catch it, it's not enforced.
5. Pre-commit + CI form a two-layer gate. Pre-commit gives fast feedback; CI is the enforcement of record.
6. Lint config (.golangci.yml) is the executable form of the style guide. If it can't be linted, write a custom check or accept the drift risk explicitly.

## 2026-04-29 — RFC decision flow (main option + sub-questions)

### Insight

- An RFC issue with N alternatives is not enough. Every option drags 2–4 follow-on questions (error recovery strategy, identifier-naming corner cases, builtin allowlist, snippet visibility). Picking only the headline option leaves codegen blocked.
- Decisions split cleanly into **Main option** (the path) + **Sub-decisions** (the corner cases). Both must be locked before code starts.
- Sub-decisions are best surfaced as a numbered list inside the parent RFC. User answers `1 a, 2 b, ...` in one shot. Cheap for both sides.
- Locked decisions live in two places: GitHub issue body (discussion record) + local `tasks/decisions/NNNN-*.md` ADR (offline + grep-able). One source of truth is a myth when contributors work without GitHub access.
- ADR file format borrows from MADR / Sun: Status, Date, linked Issue, Decision, Rationale, Locked sub-decisions, Implementation outline, References.

### Self-rules

1. When proposing alternatives in an RFC, also enumerate the sub-questions that the chosen option will force. Don't ask the user to pick A/B/C without listing the trapdoors under each.
2. After the user answers, write the locked decision to **both** the GitHub issue (prepend a `## Decision (date)` block above existing alternatives) and a local `tasks/decisions/NNNN-*.md` ADR.
3. ADR filename uses zero-padded 4-digit prefix and stable kebab title — never reuse a number, never edit an Accepted ADR in place. Supersede with a new ADR.
4. Sub-decision rationale belongs **with the sub-decision**, not in a separate doc. A future reader hitting one bullet should find the "why" without bouncing files.
5. Code samples in ADRs are signatures + 5-line illustrations, not full implementations. Signatures don't drift; example bodies do.

## 2026-04-29 — Foundation infra landed (#95-103) + commit-msg scope tightened

### Insight

- Phase 0a landed all foundation infra in one commit: monorepo layout (#95), code style + stability docs (#96, #97), `.golangci.yml` (#98), pre-commit + commit-msg hooks (#99), release-please configs (#100), CI workflows (#101), PR template + DoD (#102), AGENTS.md master + auto-sync to `.cursorrules` and copilot-instructions (#103).
- `.githooks/commit-msg` regex enforces lowercase scope `[a-z0-9/_-]+`. Pre-Phase-0a commits used uppercase scopes like `docs(CLAUDE): ...` because no hook was installed. Those commits already on `main` cannot be retroactively fixed; CI invokes `validate-commits.sh` only on `origin/main..HEAD` (PR delta), so pre-existing history is excluded. Going forward every commit is gated.
- macOS BSD `grep` does not support `-P` (PCRE). `validate-commits.sh` uses `-Ev` (POSIX ERE) instead — the Conventional Commits regex needs no PCRE-only constructs, so semantics are preserved. CI Ubuntu runs are byte-equivalent.
- AGENTS.md sync drift guard works in two directions: editing `AGENTS.md` triggers regeneration + auto-restage; editing only `.cursorrules` or `.github/copilot-instructions.md` is rejected with a "do not edit generated AI rule files" message. Pre-commit hook + CI `agents-sync` job keep both copies aligned.
- RFC #103 specified a Go program (`scripts/sync-ai-docs.go`); we shipped bash (`scripts/sync-ai-docs.sh`) as a bridge until `cmd/sveltego` lands. Documented in the script header.

### Self-rules

1. **Hooks land in the foundation commit**, not later. Any contributor cloning post-Phase-0a runs `bash scripts/install-hooks.sh` before their first commit; CONTRIBUTING.md instructs this. If a contributor bypasses the hook, CI re-validates on PR.
2. **Pre-existing commit history is immutable.** Validate-commits scoping must use `origin/<base>..HEAD`, never the full repo history, to avoid blocking PRs on legacy bad commits.
3. **Cross-platform shell scripts target POSIX ERE**, not PCRE. macOS BSD grep + Linux GNU grep both support `-E`; only Linux supports `-P`. Same lesson applies to `sed -i` (BSD requires `-i ''`, GNU does not) — file each spelling difference as an inline comment when shipping shared shell.
4. **Auto-generated files have a header that says so.** `.cursorrules` and `.github/copilot-instructions.md` start with `<!-- AUTO-GENERATED from AGENTS.md by scripts/sync-ai-docs.sh — DO NOT EDIT -->`. Pre-commit reverse-guard rejects edits to either file when AGENTS.md is unchanged.
5. **Foundation issues close in the same commit that lands their infrastructure.** Issue close happens via `gh issue close` after `git push`, not before, so the close comment can cite the commit SHA.

## 2026-04-29 — Phase 0a CI red autopsy (CRLF + workspace-root golangci-lint)

### Insight

- First Phase 0a push went red on Windows runners with `gofumpt would reformat the following files: benchmarks\doc.go ...` (14 stub files). Root cause: Windows git defaults to `core.autocrlf=true`, rewriting `\n` → `\r\n` on checkout. gofumpt rejects CRLF. Fix: `.gitattributes` with `* text=auto eol=lf` plus per-extension overrides for `.go .mod .sum .sh .yml .yaml .json .md .bash`. `.bat .cmd` keep CRLF; binaries marked `binary`.
- Same push went red on Ubuntu with `golangci/golangci-lint-action@v6` exit 7: `pattern ./...: directory prefix . does not contain modules listed in go.work`. The action runs `golangci-lint run ./...` from `$GITHUB_WORKSPACE`, which is `go.work` root with no `go.mod`. workspace-root invocation is unsupported.
- Replaced action with manual install + per-module loop iterating `go list -m -f '{{.Dir}}'`, sharing the same path-aware skip pattern as vet/test/build steps. Action's caching benefits lost — acceptable until upstream fixes workspace handling.
- First manual install attempt failed silently: `golangci/golangci-lint info found version: 1.62.2 ...` then exit 1 with no diagnostic. install.sh writes to `$bindir` and exits non-zero if dir absent. `$(go env GOPATH)/bin` doesn't exist on fresh setup-go runners. Fix: `mkdir -p` before `sh install.sh`, plus `set -euxo pipefail` and a post-install `--version` probe to surface real causes next time.

### Self-rules

1. **Repos with Windows CI legs ship `.gitattributes` from day one.** Without it, any text file fails Linux/macOS-authored format checks the moment Windows touches the working tree. Set `* text=auto eol=lf` baseline plus extension overrides; mark images binary; keep `.bat .cmd` CRLF.
2. **Don't trust workspace-aware GitHub Actions for `go.work` repos.** Multi-module workspaces are a minority case and most tool actions assume single-module repos. Default to manual install + per-module loop; add the action back only after verifying it handles `go list -m`.
3. **`install.sh`-style scripts get `set -euxo pipefail` and an explicit verification probe.** Silent install failures waste a CI cycle. The probe (`<binary> --version`) makes the next failure mode obvious in logs.
4. **`$(go env GOPATH)/bin` is not guaranteed to exist** on fresh runners until something `go install`s into it. Always `mkdir -p` before piping a downloader at it.
5. **CI red is normal during foundation phase.** First push green is the exception, not the rule. Budget 1–2 fix cycles for any new workflow before declaring Phase 0a complete.

## 2026-04-29 — Phase 0b landed (#104 golden + #105 bench gate)

### Insight

- Golden harness lives in `test-utils/golden` as a generic `golden.Equal(t, name, got)` helper, NOT inside any codegen package. RFC #104 sketched the harness adjacent to `codegen_test.go`, but codegen doesn't exist yet — building the harness as a reusable library lets every package consume it without circular deps.
- Two-mode update toggle (`-args -update` flag AND `GOLDEN_UPDATE=1` env) covers both ergonomic local use and CI scripts. `init()`-time flag registration with a sync.Once guard avoids "flag redefined" panics when multiple packages import the harness in one test binary.
- Bench gate gets its own CLI (`benchmarks/cmd/bench-compare`) instead of the bash `scripts/bench-gate.sh` from RFC #105. Reasons: bash parsing of benchstat CSV is fragile (header rows, blank lines, multi-section format); Go gives table-driven tests for the gate itself; one extra binary on CI is acceptable.
- Trivial `BenchmarkNoop` keeps the workflow exercising end-to-end. Without it, `go test -bench=.` returns "no benchmarks" and benchstat produces empty CSV — gate would never catch a real regression at integration time.
- `bench.yml` triggers on both `pull_request` (gate) and `push: branches: [main]` (deferred baseline persistence). On main push the job is `if: github.event_name == 'pull_request'` so it shows as `skipped`. Acceptable noise vs revisiting the workflow when baseline storage lands.
- `benchstat -format=csv` requires Go 1.22+ in install path; `go install golang.org/x/perf/cmd/benchstat@latest` works against `setup-go@v5`.

### Self-rules

1. **Reusable test helpers live in their own package**, not adjacent to the consumer. `test-utils/<helper>` keeps zero coupling and lets every package import without back-references.
2. **Update toggles need both flag + env.** Local dev wants `go test -args -update`; CI scripts want `GOLDEN_UPDATE=1 go test`. Support both. Guard flag registration with `sync.Once` so multi-package test binaries don't double-register.
3. **Parsers for tool output go in Go, not bash.** When a workflow's logic depends on parsing third-party output (benchstat csv, jq results, etc.), write the parser in Go with table-driven tests. Bash wrapper scripts are fine for orchestration; never for parsing.
4. **Workflows for not-yet-built features need a smoke trigger.** A `BenchmarkNoop` (or `TestNoop`) keeps the pipeline alive end-to-end. Without it, the workflow never proves itself before real tests land — first real bench would also be the first time anyone validates the gate.
5. **Skipped jobs on triggers we don't gate yet are acceptable noise.** If `bench.yml` runs on `push: main` only to keep future baseline persistence one-line away, keep the trigger and `if:`-skip the job. Cheaper than rewriting the workflow later.

## 2026-04-29 — Phase 0c landed (#94 non-goals + ADR 0005)

### Insight

- ADR 0005 mirrors GitHub issue #94 with the locked decision block prepended above the existing draft. Issue body keeps full reasoning; ADR keeps Implementation outline pointing at where each non-goal is enforced (codegen rejects `+page.ts`, no `kit.I18n` package, etc.). Both are canonical; they don't compete because the issue is the discussion log and the ADR is the offline grep-able record.
- Auditing #94's draft caught a stale risk note: "Cloudflare adapter may flip later" — but `packages/adapter-cloudflare` already exists in the workspace. The non-goals doc is allergic to drift; what's listed as "out of scope" must match what's actually missing from the codebase. Fixed inline as part of the lock.
- AskUserQuestion with 4 focused sub-decisions (View Transitions, i18n + forms, Cloudflare risk note, re-eval cadence) was right-sized — answer in <30 seconds, every option mutually exclusive. Earlier RFC locks (e.g., #1-4) used larger interviews; #94 just needed gap-fill on a substantially-drafted body.
- Three orthogonal docs (`tasks/todo.md` "Out of scope", `CLAUDE.md` "Out of scope (do not propose)", `README.md` "What it is not") all carry copies of the non-goal list. Drift between them was real — `README.md` was sparser than `CLAUDE.md`, both lagged the new ADR. Single-pass cross-doc edit kept them aligned. Cross-doc consistency rule (CLAUDE.md §12) earned its keep.

### Self-rules

1. **When locking an RFC that already has a substantial draft, AskUserQuestion only on the gaps.** Don't re-interview categories the user already wrote out. Burden of proof is on the new sub-decision (View Transitions, i18n, etc.), not on what's already drafted.
2. **Audit "may flip later" notes against current codebase before locking.** A non-goals doc that contradicts shipped packages is worse than no doc — readers stop trusting it. Run `ls packages/` and check workspace before locking.
3. **Issue + ADR together; never one without the other.** Issue body holds discussion record (above-the-fold Decision block + original sub-options as history). ADR holds offline reference + Implementation outline. The Implementation outline is the unique value of the ADR — names the codegen rejection point, the missing package, the reading direction.
4. **Cross-doc copies of canonical lists need a single-pass sync rule.** When `tasks/todo.md`, `CLAUDE.md`, `README.md` all carry their own copy of "out of scope", reduce to: ADR is canonical, others get a one-line cross-ref + short bulleted summary. Don't maintain three full copies.
5. **`gh issue edit --body-file` via `gh issue view --json body --jq .body`** is the safe round-trip for editing a long issue body. `--body` inline blows up on quoting; `--body-file -` from stdin works but loses the round-trip safety of editing on disk first.

## 2026-04-29 — Phase 0d landed (cobra CLI bootstrap)

### Insight

- RFC #5 was authored pre-monorepo with single-module `cmd/`, `internal/`, `pkg/` layout. RFC #95 (Phase 0a) replaced that. Phase 0d delivers the CLI inside the monorepo path `packages/sveltego/cmd/sveltego/` — same files, different parent. The reconciliation belongs in the close comment, not in a re-write of the issue body.
- **Cobra command vars carry state across `Execute()` calls.** Reusing a package-level `var versionCmd = &cobra.Command{...}` across tests means the second test inherits flags parsed by the first. Factory pattern (`newVersionCmd()` returns a fresh tree, `NewRootCmd()` calls all `newXxxCmd()`) eliminates the bleed by construction. Tests should always call `NewRootCmd()`, never reach for the package-level `RootCmd`.
- **`runtime.Version()` already returns `"go1.22.5"` with the prefix.** Spec saying `fmt.Fprintf(..., "go%s", runtime.Version())` produces `gogo1.22.5`. Strip the prefix in a helper. Caught only because tests asserted regex `^sveltego v\S+ \(go\d+\.\d+, ...\)$`.
- **slog has no public level getter.** Tests asserting verbosity flag effect must call `slog.Default().Enabled(ctx, slog.LevelDebug)` rather than introspecting the handler. Equivalent observable signal, no reflection.
- **release-please workflow fails by default on new repos:** `GitHub Actions is not permitted to create or approve pull requests` from `googleapis/release-please-action@v4`. Two fixes: (a) Settings → Actions → General → Workflow permissions → enable "Allow GitHub Actions to create and approve pull requests"; (b) add `permissions: pull-requests: write, contents: write` to the workflow file. Not blocking — CI separate from release-please. Logged for follow-up.
- **LSP cache lag fires misleading diagnostics post-`go mod tidy`.** When agent adds a new dep (cobra) and `go mod tidy` rewrites go.sum, the editor's gopls cache may report `BrokenImport` for ~30s while it re-indexes. Independent `go build` from shell confirms the actual state. Don't trust LSP diagnostics if they contradict a clean shell build.

### Self-rules

1. **Cobra subcommands as factories**, never as package vars referenced from tests. Pattern: `newXxxCmd() *cobra.Command` for each, `NewRootCmd()` composes them. Package var only for the binary entry (`var rootCmd = NewRootCmd()` consumed by `Execute()`).
2. **`runtime.Version()` includes the `"go"` prefix.** Strip it before formatting alongside literal `go` in user-visible strings. Same applies to `runtime.GOOS` / `runtime.GOARCH` (no prefix; safe to concatenate).
3. **slog tests assert observable behavior**, not handler internals. `slog.Default().Enabled(ctx, level)` is the contract.
4. **When LSP and shell disagree, trust the shell.** Re-run `go build` / `go vet` / `go test` from a fresh shell session if diagnostics report imports the disk obviously has. Don't immediately edit code to "fix" a stale-cache symptom.
5. **release-please needs repo settings + workflow permissions.** Either Settings UI or `permissions:` block in the workflow. Document both; pick one per project. Without it, every push to main fails the release-please job silently — easy to miss when CI proper is green.

## 2026-04-29 — Phase 0e landed (parser foundation)

### Insight

- **Issue specs lock package paths the user may override.** Issues #7 and #8 both said `internal/parser/` for token/lexer/AST/parser. User's locked layout decision was three-package split (`internal/lexer/`, `internal/ast/`, `internal/parser/`). The agents need an explicit "override the issue" instruction in the brief — agents will otherwise default to the issue body and produce a single fat package.
- **Field/method name collisions when a struct field shares a getter name.** Issue #8 design showed both `Pos Position` field and `Pos() Position` method on every node. That collides in Go. Phase 0e-B agent renamed the field to `P` and kept `Pos()` as the accessor. Mechanical resolution but it ripples — every parser construction site uses `&ast.Mustache{P: ast.Pos{...}}` not `&ast.Mustache{Pos: ast.Pos{...}}`. Brief downstream agents about it explicitly; they will otherwise transcribe the issue spec verbatim and fail to compile.
- **ADR sub-decisions can defer too aggressively.** ADR 0001 deferred multi-error model to "post-MVP Phase 2". Issue #8 acceptance criteria required ≥2 errors per multi-mistake input — i.e., the deferred work was already MVP-blocking. Reading the dependent issue's acceptance list before sealing the ADR would have caught it. Caught at Phase 0e launch instead.
- **LSP DuplicateDecl false positives after agent file splits.** When an agent moves helper functions from `parser.go` into a new `helpers.go`, gopls reports DuplicateDecl on both files for ~30s while re-indexing. Independent `go build` + `go test` from shell shows clean. Same class as the cobra `BrokenImport` lag from Phase 0d. Reinforces the "trust shell over LSP" rule.
- **Multi-error parser test as forcing function.** A single fixture with two distinct mistakes is the only way to prove recovery actually re-syncs and continues. Without that fixture, a parser that silently aborts on the first error passes every other test. Make the multi-error test a Phase 0e acceptance gate, not a Phase 1 nice-to-have.
- **`text/dump` over JSON for AST goldens.** JSON serialization of an AST with interface fields can't fully round-trip without custom unmarshalers — the discriminator is lost. Indented text dump (`Element "div" @1:5\n  Attribute "class" static="foo" @1:10`) sidesteps the unmarshal problem and makes diffs readable for human review of `-update` flows. JSON acceptance criterion in issue #8 was an over-spec for the actual goal (deterministic diffable output).
- **Two parallel agents on touching layers (lexer + ast) work because each defines its own Pos.** Agent A's Token has flat `Offset/Line/Col` fields. Agent B's `ast.Pos` is a struct. Parser bridges by reading lexer fields and constructing `ast.Pos`. No cross-import, no cycle. The cost is field-naming friction at the bridge, not at the package boundary. Worth it for parallelism.

### Self-rules

1. **Always state package-layout overrides explicitly in agent briefs.** "Issue #N puts files at path X; override to path Y, do not file files at X" — verbatim. Otherwise agents follow the issue body and the orchestrator pays a fix-up round.
2. **Field-vs-method name collisions on AST nodes default to renaming the field, never the method.** Method is the interface contract (`Node.Pos()`); field is internal storage. Brief downstream agents on the field name once; cite the file (`packages/sveltego/internal/ast/nodes.go`) rather than re-declaring the convention each time.
3. **Read every dependent issue's acceptance criteria before locking an ADR sub-decision.** "Phase 1 / Phase 2" deferrals are valid only if the dependent MVP issues don't already require Phase 2 behavior. Otherwise the deferral is fictional and the ADR misleads.
4. **Trust shell over LSP** (reinforced from Phase 0d). When LSP reports DuplicateDecl, BrokenImport, or any errors that contradict `go build`, run the shell and move on. Never edit code in response to a diagnostic without a confirming shell signal.
5. **Multi-error parser tests are mandatory, not optional.** A single fixture with ≥2 distinct mistakes proves recovery sync. Add it to the test plan up front, not as a follow-up.
6. **AST goldens use indented text dump, not JSON.** JSON loses interface discriminators. Text dumps diff cleanly. Reserve JSON for serialization to the wire, not for in-repo goldens.
7. **Cross-module imports need BOTH go.work entries AND go.mod require+replace.** Workspace mode (`go.work`) is the dev-time cross-module resolver. Isolated mode (`GOWORK=off` — used by the `isolated-modules` CI job per RFC #95) requires the consuming module's `go.mod` to carry `require <module> v0.0.0-00010101000000-000000000000` plus `replace <module> => <relative-path>`. First cross-module consumer pays the wiring cost; subsequent consumers reuse it. An agent that says "no go.mod change needed" without verifying GOWORK=off is wrong by default — always run `GOWORK=off go build ./... && GOWORK=off go test -race -short ./...` from the consuming module before declaring any cross-module wiring complete.

## Phase 0f — render + kit + codegen pipeline (2026-04-29)

### Insight

- **Render API drift between ADR draft and shipped code.** ADR 0004 listed `WriteAttr(name, val string)` as part of the runtime surface. The shipped `render` package never grew that method — codegen composes attributes inline as `WriteString(\` name="\`) + WriteEscapeAttr(val) + WriteString(\`"\`)`. Root cause: the ADR was written before the reductive realization that quoting context belongs in codegen, not runtime. The doc-as-spec lagged the design. Phase 0f wrap caught the drift only because the smoke test forced a re-read of the actual render surface.
- **`restoreRunesBytes` round-trip placeholder is a smell, not a feature.** Phase 0f-E ships codegen that rewrites `$rune` → `__sveltegoRune__` to keep `go/parser` happy, then `restoreRunesBytes` un-substitutes after `go/format`. The output does not compile while the rune reference is intact. That is rune-lowering scope creep leaking into the script-extraction phase. The issue body for #14 said "stash for downstream", and we stashed plus transformed plus restored. Should have stashed only.
- **Struct-literal-only PageData was a deliberate user pivot off the issue body.** Issue #15 discussed inferring from explicit `type PageData struct{...}` declarations in `+page.server.go`. The user picked struct-literal-only on `Load()` return as the MVP rule. The ADR amendment captures both the chosen path AND the rejected one (explicit-type) so a future reader does not re-litigate the deferral.
- **LSP false-positive on `restoreRunesBytes`** (UndeclaredName at `codegen.go:82` despite the func existing in `script.go`) reinforced the existing rule (Phases 0d/0e): trust shell `go build` output, not LSP cache, for cross-file symbol lookups. Same class of stale-index bug we have now seen three phases running.

### Self-rules

1. **When an ADR lists API methods, regenerate the list from `grep '^func' <package>/*.go` at codegen time, not from the ADR draft.** Doc-as-spec is fine for capture; doc-as-source-of-truth is wrong once code lands. Every ADR with a code-shaped block gets a "regenerate from disk" step before declaring it locked.
2. **When an issue body says "stash for downstream", stash only — do not transform or round-trip.** No source rewriting in the phase that defers the feature. The placeholder pass for runes belongs deleted once #43–#47 land; file the followup now so the cleanup is tracked, not folklore.
3. **When the user picks a sub-decision that contradicts an issue body, the ADR amendment must capture both paths.** Chosen path goes into the locked sub-decision bullet; rejected path goes in the same bullet as a one-clause "out of scope until <future RFC>" note. Future Claude reads one bullet and gets the full lattice without bouncing files.
4. **Smoke tests for codegen-style pipelines pick fixtures by what compiles, not by what is named first.** Fixtures referencing unbound identifiers (`Data.Ok`, `Items`) need either companion declarations or a `+page.server.go` to be valid. State the binding strategy in the test up front; never assume "it parses, so it builds".
5. **LSP cross-file symbol lookups stay on the bench until shell agrees.** When LSP reports `UndeclaredName` for a symbol declared in a sibling file in the same package, run `go build` and ignore the diagnostic. Never edit functioning code to silence a stale-index warning.

## Phase 0g — router foundation (2026-04-30)

### Insight

- **Sequential dispatch (A → B → C) was the right call vs forced parallelism.** Phase 0g-A shipped the runtime radix tree + matcher API; B layered the route scanner on A's exported `Segment` / `SegmentKind`; C bolted the manifest emitter and `kit/params` built-ins on B's `ScanResult`. Forcing parallel runs would have required a phantom "types-only" sub-phase whose only output was a frozen interface header — cheap to write, but the orchestration cost (extra agent context, extra rendezvous) exceeds the wall-clock saving on a job this small. Sequential beats parallel when each layer's deliverable is narrow and the next layer's first move is to import the previous one.
- **LSP false-positive class continues across phases.** Cross-file undefined-symbol claims (`encodePackageName`, `builtinMatchers`, `discoverMatchers`) appeared during Phase 0g-B/C any time the LSP cache was stale right after a multi-file commit. Phase 0d (cobra), 0e (parser), 0f (codegen) all hit the same pattern. The shell `go build` + `go test -race` are authoritative; the LSP cache is advisory at best for the 30-second window after a batch lands.
- **Pre-commit `golangci-lint --new-from-rev=HEAD ./...` failed at workspace root** because the linter cannot process modules with mixed go.work / GOWORK=off state from a single invocation. Fix in the hook: scope per-module via the same `go list -m -f '{{.Dir}}'` loop that CI uses. This caught me twice now (Phase 0a CI fix + Phase 0g-C local hook fix); the rule belongs in the hook code, not in lessons.
- **`strconv.Atoi("+1")` accepts the leading sign.** The `int` matcher's first cut accepted `+1`, `-0`, and other sign-prefixed inputs — fine by the function name, surprising for the spec's intent of "positive integer-looking string." We kept Atoi semantics for MVP and documented the surprise; a stricter `posint` matcher can ship later. The general lesson is that built-in matchers need a published truth table (positive examples + negative examples + edge cases) at the moment they ship, not in a follow-up PR.
- **Manifest emitter's `Page{}.Render` invocation pattern is load-bearing on ADR 0004.** The codegen-emitted package shape (struct `Page` with method `Render`) is what the manifest cross-imports. ADR 0004's PageData inference therefore has a forward-reference into ADR 0003's manifest emit; ADR 0003's amendment for `GenerateManifest` has a backward-reference into ADR 0004's `Page{}.Render` shape. Both ADRs must point at each other so a future reader hitting one finds the other in one hop.

### Self-rules

1. **Choose sequential dispatch when each phase's first action is to import the previous phase's exports.** The dependency arrow is the orchestration constraint; a "types-only" sub-phase is cheaper to skip than to add. Save parallelism for phases whose interfaces are already locked from a prior milestone.
2. **When LSP reports `unusedfunc`, `UndeclaredName`, or `DuplicateDecl` on a symbol declared in a freshly-landed batch, run `grep -rn '<symbol>' <package>/` before reaching for an editor.** Three phases of false positives establishes the pattern. Do not modify code to silence an LSP diagnostic that contradicts a clean shell build.
3. **Pre-commit hook gates that work in CI must be tested at workspace root with multi-module GOWORK=off before merging.** The pattern is "for each `go list -m -f '{{.Dir}}'` entry, run the gate inside that directory." Any hook that runs lint/vet/test from `$(git rev-parse --show-toplevel)` against a `go.work` parent fails the moment a contributor lands an isolated-module change. Fix the hook, don't fix the workflow.
4. **Every kit/params built-in matcher ships with a truth table.** Truth table covers `+`, `-`, leading zeros, empty string, whitespace, mixed case, the canonical form, and at least one Unicode boundary case. Truth table goes in the matcher file as a comment block adjacent to the package var; tests exercise every row. Not optional — matchers are user-visible API and surprise here is expensive.
5. **When an ADR amendment lands, grep for cross-references in other ADRs and add forward references both ways.** ADR 0003 amend ⇄ ADR 0004 amend is the working example: 0003's manifest emit references 0004's `Page{}.Render`; 0004's PageData inference is consumed by 0003's emitter. One-way pointers rot the moment the consumer is the side that gets reread.

## Phase 0i-fix — three framework bugs at MVP boundary (2026-04-30)

### Insight

- **Empirical-validation rule for spec claims.** ADR 0003 locked the SvelteKit-style `+page.server.go` filename and the `[slug]/` directory shape on the assumption Go would tolerate both. Five minutes with `go list ./...` against a real fixture would have rejected both at the time the ADR landed. Both bugs sat latent for months and surfaced only at the Phase 0j hello-world smoke. Rule: every "we just need X" assumption in an ADR earns a five-minute toolchain probe before the ADR seals.
- **Mirror tree pattern for "user code Go cannot load directly".** The `[slug]/` directory is invalid as a Go import path (the `[` character is illegal in module paths). Renaming the directory breaks SvelteKit parity. The fix is decoupling: user owns the source path; codegen owns a Go-loadable mirror at `.gen/usersrc/<encoded>/` whose directory and filenames are deterministic underscores. The wire glue imports the mirror, never the user tree. Authoring DX and compiler constraints stop fighting once each side has its own path.
- **`any`-widening adapters are the right shape when a strongly-typed user method must satisfy a `func(any)` interface field.** Manifest emit cannot assign `Page{}.Render` (typed `data PageData`) to `router.PageHandler` (typed `data any`). Renaming the runtime field destroys the type-erasure contract; widening the user method destroys the user's type-safety. The middle ground is a per-route adapter function emitted by codegen that does the assertion and forwards. The user keeps PageData; the runtime keeps `any`; the adapter is the only place the cast happens, and a mismatch fails fast with a descriptive `fmt.Errorf`.
- **Build constraint `//go:build sveltego` is the cheapest possible exclusion mechanism that keeps gopls / golangci-lint quiet on non-loadable trees.** Cheaper than rename-and-rewrite, cheaper than sub-module schemes, cheaper than `_`-prefix file naming (which still gets caught by gopls's "underscore_prefixed file" walk in some scenarios). The only surface area is one comment line at the top of every user `.go` file plus a scanner diagnostic when it's missing.
- **Go validates filenames before honoring build tags.** A file named `+page.server.go` is rejected with "invalid input file name" regardless of any build constraint inside the file — the package list step happens before file content is read. Renaming is the only fix; tagging cannot rescue a file Go won't even open. By contrast, a file named `page.server.go` carrying `//go:build sveltego` *inside* a `[slug]/` directory works: Go opens the file, reads the constraint, decides to exclude it, and the directory ends up with zero compilable Go files — at which point Go silently skips the directory instead of validating its name.
- **Manifest references symbols in the gen package; the gen package must declare them.** ADR 0003 sketched a `wire.gen.go` re-export but no codegen step actually wrote one — manifest emitted `<gen>.Load` / `<gen>.Actions` against an empty package. Two-side contracts where one side is generated and the other is not need an explicit emitter step. "It's mentioned in the ADR" is not a substitute for running the codegen and reading the diff.
- **`Actions()` stubs are cheaper than conditional manifest emission.** When the user's `page.server.go` declares Load but no Actions, the wire could either emit nothing for Actions (forcing the manifest to know which routes have Actions) or emit a `func Actions() any { return nil }` stub. Stubbing keeps the manifest emitter ignorant of per-route detail and removes a coordination point that would have been a follow-up bug.

### Self-rules

1. **Probe toolchain assumptions before sealing an ADR.** When an ADR locks a filename or directory shape, run `go list ./...` against a fixture matching the proposal before declaring it accepted. The probe takes five minutes; missing it costs a Phase boundary's worth of fix-up.
2. **Mirror trees for user source whose path Go cannot load.** When a user-facing path convention conflicts with Go import path rules, decouple via a deterministic mirror at `.gen/usersrc/<encoded>/`. User owns the source path; codegen owns the loadable copy. Never rename the user tree to please the toolchain.
3. **`any`-widening adapters belong in codegen, not in the runtime contract.** When a generated typed method must satisfy a `func(any)` interface field, emit a per-route adapter function that does the type assertion and forwards. Do not weaken the user method; do not widen the runtime field; emit the adapter.
4. **Every user `.go` file under sveltego conventions starts with `//go:build sveltego`.** Scanner emits a warning diagnostic when the line is absent; codegen drives `go/parser` directly so the missing constraint does not break codegen, only `go build` from outside.
5. **When a generated file references symbols, an emitter step writes those symbols.** "Mentioned in the ADR" is not "implemented in the pipeline." Every cross-package symbol reference in a generated file traces back to a generator function with a name and a test; no exceptions.
6. **Stub before conditional emission when the cost is one no-op function.** A `func Actions() any { return nil }` stub keeps the manifest emitter ignorant of per-route detail. Conditional emission is a coordination point waiting to misfire.
7. **Filename validation in Go runs before build tags.** A file named with an invalid character (`+`, `:`, etc.) cannot be rescued by `//go:build ignore` because Go rejects the filename before opening the file. Plan filename schemes against this rule, not against the build-tag escape hatch.

- **Inline-struct-literal Load + named-PageData adapter is a runtime type-mismatch by construction.** Phase 0i-fix's pipeline emits `type PageData struct{...}` in the gen page package and a per-route adapter that does `data.(<gen>.PageData)`. ### Self-rules

1. **Every codegen feature with a runtime contract gets a black-box smoke before declaring it landed.** File-content assertions catch emit-time bugs; only `go run` catches type-assertion mismatches, ABI drift, and adapter wiring bugs. The smoke fixture lives in a playground or `-tags=integration` test; a unit test that compares emitted strings is necessary but not sufficient.
2. **Modules whose `cmd/*` imports generated packages get an explicit skip in the per-module CI loop, paired with a dedicated codegen-aware job.** The skip is a structural requirement, not a temporary workaround. Document the reason inline so a future maintainer does not "clean it up" without realizing why it exists.
3. **When user-facing convention requires a particular Load shape (inline struct literal here), test that the shape compiles AND runs end-to-end before sealing the convention.** ADR 0004's struct-literal-only PageData rule looked clean on paper; only the runtime smoke proved that the rule plus the adapter emitter cannot interoperate. Convention reviews must include a runtime hop, not just a parse-and-emit hop.
4. **Phase boundary discipline: when a smoke fails because of a framework bug, file the issue, document the partial state, and stop.** Do not silently fix the framework from inside the playground phase — the fix touches a different package and deserves its own commit, its own test, and its own ADR amendment if needed. Filing #109 plus updating `playgrounds/basic/README.md` with the blocker is the correct partial-completion state.

## Phase 0j-fix — MVP closure (2026-04-30)

### Insight

- **Type alias vs new named type for inferred PageData is a one-character bug with runtime-only consequences.** Phase 0i-fix emitted `type PageData struct{...}`; this looks identical to `type PageData = struct{...}` until `data.(PageData)` runs against an anonymous struct literal. New named types break type identity (anon-to-named is a value conversion, not assertion); aliases preserve identity. The build-test suite passed because it asserts on emitted strings, not runtime semantics. Hello-world smoke surfaced it on the first request, just like #109's filing predicted. Two-character fix (`type X struct` → `type X = struct`); 60+ goldens regenerated; manifest adapter and wire emitter unchanged.
- **Pre-commit hook skip pattern for non-loadable user code splits cleanly into universal-formatting vs lint+test.** Universal formatting (gofumpt, goimports) cares about source bytes only — it works on any `.go` file Go's parser accepts, regardless of build tags or directory shape. Lint and test require a loadable package, which fails on bracketed paths and all-tagged dirs. The fix is a single filter applied between the formatting step (over `staged_go`) and the lint+test step (over `testable_go`). Filter has two layers: path-based (catches `src/{routes,params}/` and `hooks.server.go`) plus build-tag-based (catches any `.go` with `//go:build ... sveltego ...` first line, defense-in-depth for files outside the convention paths).
- **macOS bash 3.2 rejects `case` statements inside `while IFS= read` inside `$(...)`** with "syntax error near unexpected token `;;`" even when the syntax is valid bash 4+. Workaround is to use chained `grep -vE` filters for path-shape skips (`grep` is portable across bash versions) and reserve the `while read` loop for the build-tag content check that genuinely needs file I/O. Two `grep -vE` invocations plus one `while read` loop is uglier than a single pipeline but works on every shell the repo claims to support.
- **MVP-end retrospective: Phase 0j surfaced 5 framework bugs over 3 fix passes (#106–108 in 0i-fix, #109–110 in 0j-fix).** All five were latent from earlier phases; none were caught by unit tests. The forcing function in every case was end-to-end smoke (`go run ./cmd/sveltego compile && go build && curl`). File-content assertions in `internal/codegen` test packages caught zero of the five. The pattern is unambiguous: codegen features that compose into a runtime contract need a black-box smoke before declaring landed, not after MVP close.

### Self-rules

1. **When generating a struct type to receive an inline anonymous-struct literal, prefer the type alias form (`type X = struct{...}`) over a new named type.** The alias preserves runtime type identity for the assertion path; the new type forces a value conversion that the wire layer must emit explicitly. Default to the alias unless the codegen has a concrete reason for a distinct nominal type (e.g., method receivers).
2. **Pre-commit hook filters that target a specific file convention belong as path-based `grep -vE` (portable, bash 3.2-safe) plus a `while read` build-tag check (only when path filter is insufficient).** Universal-formatting steps stay outside the filter — gofumpt and goimports work on any parseable `.go` file. Lint and test steps consume the filtered list.
3. **End-to-end smoke is the forcing function for codegen-runtime contracts; schedule it once per MVP-class feature, not once at MVP close.** Five latent bugs across two fix passes is direct evidence the unit tests don't cover the runtime contract. Add a `go run` or HTTP-curl smoke as soon as a codegen step claims to produce a loadable package; do not wait for the integration milestone.
4. **When a fix changes a hot-cache emit shape (here, every `page.gen.go`), regenerate goldens with `GOLDEN_UPDATE=1` and inspect the diff line-by-line before committing.** 60 golden files flipped from `type PageData struct` to `type PageData = struct`; the diff is large but trivially uniform. Visual inspection is cheap insurance against an emit drift that the test runner might silently accept.

## Phase 0k-A — layout chain rendering (2026-04-30)

### Insight

- **Go's build system silently excludes filenames starting with `_`.** A first emit of `_layout.gen.go` compiled stand-alone via `go build ./.gen/routes/` because Go was already in the directory and didn't apply the underscore filter, but `go list ./.gen/routes` returned `[page.gen.go wire.gen.go]` — the file was simply ignored as if missing. The manifest in the parent package then failed with "undefined: page_routes.Layout" even though the symbol clearly existed in the underscore-prefixed file. Lesson: any `_`-prefixed name is invisible to the toolchain regardless of build tags. The fix was renaming to `layout.gen.go`.
- **Breaking changes to runtime types (`router.Route.LayoutChain` from `[]*Route` to `[]LayoutHandler`) are cheap pre-stable but demand grep across the entire repo.** The old field was unused, so the change was source-compatible with every consumer — but only because no one had wired it. Future changes to stable runtime types need full reference enumeration before flipping. The `[]*Route` shape was a Phase 0h placeholder; documenting the eventual handler shape from day one would have prevented this churn.
- **Slot lowering depends on whether the enclosing emitter is a Page or a Layout.** A naïve `<slot />` → `children(w)` lowering fails to compile inside a Page (no `children` parameter). The minimum-cost solution is a Builder flag (`hasChildren`) that the slot emitter consults; layouts set it true, pages leave it false and emit a TODO comment. Thread emitter context through the Builder, not through every emit function signature.
- **Layout adapter emission and route adapter emission share the alias pool.** A layout dir that also owns a +page.svelte resolves to the same gen package; both adapters import it via the same alias. Tracking aliases per-package (rather than per-route or per-layout) yields a single import line. Emitting two distinct aliases for the same package would compile but produce duplicate import lines and break gofumpt's import grouping.
- **Layout pipeline composition reverses the chain: outer wraps inner, inner is innermost.** Iteration runs `for i := len(chain)-1; i >= 0; i--` to build closures from inside out. The resulting `inner` closure, when called, runs the outermost layout, which calls its `children`, which is the next-out layout, and so on until the page. Order of iteration matches the chain encoding (ancestor → self) but inverts during composition.

### Self-rules

1. **Generated filenames must not start with `_`.** Go's build tooling ignores them silently — no warning, no error, just a missing symbol downstream. Use `layout.gen.go`, `wire.gen.go`, etc. The same rule applies to `.`-prefixed names. Prefer descriptive prefixes (`layout_`, `page_`) over leading separators.
2. **Verify generator output via `go list -f '{{.GoFiles}}'` on the emitted package.** A successful `go build ./<pkg>/` is not proof of a complete file set; the toolchain may have silently filtered files. `go list` reports the canonical accepted set and surfaces invisible files immediately.
3. **Every emit step that produces a file gets a `go list` smoke in its build test.** Cheap insurance against filename rules (`_`, `.`, `testdata/`), build-tag exclusions, and casing mismatches that produce a "missing symbol" error far away from the real cause.
4. **When a Builder needs context-dependent behavior (e.g., slot lowering varies by Page vs Layout), thread the flag through the Builder struct, not through every emit function signature.** The flag is set once at the top of the generator; emit code paths read it without modification.
5. **Pre-stable runtime type changes must enumerate every reference site before flipping.** The change is cheap source-wise but a missed consumer is a silent runtime bug. `grep -rn '<TypeName>' packages/` plus a clean compile across all `go.work` modules is the minimum bar.

### Insight

- **Issue #33 specced `panic(RedirectErr{...})`; we shipped error returns instead.** SvelteKit's `throw redirect()` is JS-flavored; Go's idiom is `return val, err`. Panic-as-control-flow is a known anti-pattern outside of unrecoverable bugs — it interferes with stack-trace analysis, complicates `defer/recover` chains in middleware, and forces every Load callsite to live behind the pipeline's recover guard. Error returns let `errors.As` route the sentinel cleanly, keep the Load contract `(any, error)`, and stay grep-able. The pipeline already had an `httpStatuser` interface lookup; sentinels slot in as additional `errors.As` checks before the generic interface fallback.
- **Branch order in `handleLoadError` matters because `RedirectErr` also implements `httpStatuser`.** The old `httpStatuser` fallback would have grabbed the redirect status (303) and called `writePlain` — losing the Location header entirely. Sentinel-typed checks must run before the generic interface lookup; otherwise a sentinel that happens to satisfy a broader contract gets misrouted. Same shape applies to any future sentinel: its specific branch goes first, generic interfaces fall through last.
- **sloglint's `kv-only: true` plus `no-raw-keys: true` rejects ad-hoc `slog.String("location", ...)` calls.** The repo enforces a closed set of `logKey*` constants used as kv-pair keys. Mixing `slog.Int(...)` with kv-style args also trips `no-mixed-args`. Adding new log attributes requires (1) new constant in the per-package log key block, (2) kv-pair usage at the callsite. Dropping a `slog.X(...)` builder call into otherwise kv-style code fails lint silently from a unit-test perspective — only `golangci-lint run` catches it.
- **`http.Redirect` writes the Location header AND the status code.** No need to call `w.Header().Set("Location", ...)` separately. The function also handles the response body (writes a small HTML stub when method permits). For 303/307/308 with no body content needed, `http.Redirect` is the one-call API.

### Self-rules

1. **Prefer error returns over panics for control-flow sentinels in Go.** When porting a JS framework that uses `throw` for short-circuits, translate to typed errors plus `errors.As` at the boundary. Document the departure from upstream spec inline (godoc on the helper) and in lessons. The only exception is genuinely unrecoverable bugs where the program cannot continue.
2. **Sentinel-type checks in error-handling chains run before generic-interface checks.** When a sentinel implements both a specific identity and a broad interface, the broader interface lookup must come last or the sentinel's specific behavior (e.g. Location header for redirect) gets lost. Order: specific types via `errors.As(&specific)`, then interfaces via `errors.As(&iface)`, then default fallback.
3. **Adding a log attribute requires a `logKey*` constant before the callsite.** sloglint's `no-raw-keys` plus `kv-only` is strict; raw string keys and `slog.X(...)` builder calls both trip it. New attributes get a constant in the package's log-key block and kv-pair usage at every callsite.
4. **Run `golangci-lint run` after any change that touches log calls or imports.** Local gates (`gofumpt`, `go vet`, `go test`) accept code that lint rejects; the lint gate is the only one that catches sloglint violations and mixed-args bugs.

## Phase 0p — CI budget split (PR lite vs main full) (2026-04-30)

### Insight

- **Pre-Phase-0p PR cost was ~11 jobs:** 3 OS × 2 Go = 6 lint-and-test, plus changes, isolated-modules, agents-sync, commit-lint, playground-smoke, bench. That spend is a tax on every push during iteration. Most of it (full matrix, isolated modules, playground smoke, bench) only catches regressions at merge boundaries — running them on every PR push is double-payment for the same coverage.
- **Splitting by `github.event_name` is cleaner than a conditional matrix.** Two sibling jobs (`lint-and-test-pr` with 1×1 matrix, `lint-and-test-main` with 3×2 matrix), each gated by `if: github.event_name == 'pull_request'` / `'push'`, reads top-down. A single job with `fromJSON`-driven matrix dispatch is shorter but harder to debug — when a Windows-only failure trips, the YAML reader has to mentally evaluate the `fromJSON` to know which leg ran.
- **Concurrency cancel-in-progress is a free 30% on multi-push PRs.** Without it, force-pushes during iteration queue extra runs that the next push will obsolete. With `cancel-in-progress: true`, only the latest commit's run survives; the in-flight stale runs are killed before they consume their budget.
- **`paths-ignore` short-circuits the workflow before any job spins up.** Docs-only commits (`**.md`, `tasks/**`, `tasks/decisions/**`, `docs/**`) trigger zero jobs. The gate runs at the workflow level, not the job level — no runner is provisioned.
- **bench.yml has no good reason to run on PRs.** PR bench compares base vs head, posts a sticky comment, and gates merge — but the bench-compare gate is advisory in our setup (`bench-regression:` PR-body opt-out exists), and the data lands as a comment that nobody reads on most PRs. Moving bench to push-to-main + nightly schedule + workflow_dispatch keeps the regression gate at merge boundary (where it actually blocks bad merges) and lets nightly drift detection run unattended. Manual dispatch covers the rare PR that needs explicit perf signoff.
- **Removing the bench PR trigger required dropping `github.base_ref` and `github.event.pull_request.head.sha` references** because those expressions are undefined on `push` / `schedule` / `workflow_dispatch` events. The replacement compares `HEAD~1` vs `HEAD`, which is what main-merge regression detection wants anyway.

### Self-rules

1. **CI matrices split by trigger, not collapsed.** Lite PR matrix (1 OS × 1 Go) and full main matrix (N OS × M Go) live in separate jobs gated by `if: github.event_name == ...`. The duplication is annoying once but pays back every PR. A conditional matrix via `fromJSON` is acceptable only when the matrix axes are identical and only the values differ.
2. **Every workflow gets `concurrency: cancel-in-progress: true`.** Free spend reduction on iterative pushes. The group key is `<workflow>-${{ github.ref }}` so different branches don't collide.
3. **Docs-only paths go in `paths-ignore`.** `**.md`, `tasks/**`, `tasks/decisions/**`, `docs/**`, `.github/ISSUE_TEMPLATE/**` should never trigger CI. The list lives at the workflow `on:` level so the gate runs before any job dispatch.
4. **Bench gates on push-to-main, not PR.** PR bench is advisory at best; main-push bench is the merge-time regression check. Add `schedule` (nightly cron `0 3 * * *`) for unattended drift detection and `workflow_dispatch` for manual reruns. PRs that genuinely need bench data run it locally or via dispatch.
5. **Workflow YAML changes ship in a single-scope commit (`ci(workflows):` or `ci:`).** The pre-commit `commit-msg` regex enforces lowercase scope; mixing CI YAML with non-CI changes splits the diff and trips the validator. One PR per workflow restructure keeps the diff reviewable.

## Phase 0q — hooks group (#26 + #27 + #80 + #81) (2026-04-30)

### Insight

- **No generated `main.go` in this architecture.** The brief assumed a generated `main.go` would call `hooks.Init(ctx)`, mirroring SvelteKit's adapter output. Reading `playgrounds/basic/cmd/app/main.go` first showed that user code authors `main()` and wires `server.New(server.Config{...})` itself; codegen produces only `.gen/`. Right answer: expose `gen.Hooks() kit.Hooks` from codegen, accept `Config.Hooks`, and run `Init` inside `Server.ListenAndServe` (and an explicit `Server.Init(ctx)` for callers managing startup themselves). One factory call from user main.go threads everything; nothing is generated outside `.gen/`.
- **Handle wrapping the entire pipeline forces a buffered Response model for page routes.** The pre-0q pipeline wrote directly to `http.ResponseWriter` from the render buffer — no intermediate Response value. SvelteKit's Handle returns a Response, post-modifying headers and (theoretically) the body. To preserve that contract for page routes, `renderPage` now returns a `*kit.Response` carrying status/headers/body; the pipeline writes once at the very end. +server.go handlers cannot fit this model (they use raw `http.HandlerFunc` and stream through w directly), so resolve returns `errServerRouteWrote` for them and the writer step is skipped — Handle on +server.go routes can short-circuit and inspect Locals/URL but cannot post-modify headers. Documented as MVP scope.
- **Sentinel branch order in handlePipelineError matters.** RedirectErr / HTTPErr / FailErr / SafeError must each be tried via `errors.As` *before* delegating to the user's HandleError. Otherwise a user error implementing both a sentinel-shape and the broader interface gets misrouted (e.g. RedirectErr also implements HTTPStatus → without ordering it would lose the Location header). Same lesson as Phase 0m-X2; reaffirmed for the multi-branch HandleError path.
- **Backward-compat for the legacy `httpStatuser` interface.** Pre-0q tests (`TestServeHTTP_loadError`) asserted that any error type exposing `HTTPStatus() int` drove the response status without a sentinel. The new HandleError path collapses every uncaught error into a 500 SafeError by default (identity hook). To keep that pre-existing contract green, when the SafeError comes back as the canonical 500/InternalServerError pair AND the original error implements httpStatuser, the writer promotes the status from the interface. Surgical: only the default-identity case takes the legacy promotion; user-authored HandleError that returns a deliberate 500 is left alone.
- **Panic-recovery boundary is the framework's responsibility, not the user's.** Acceptance #27 demands that "Panic in Load is caught, passed to HandleError, returns 500 with logged ID." The pipeline wraps the Handle call in `defer/recover`, lifting any `recover()` value into a wrapped error so the rest of the error-routing logic is unchanged. That's the one place the project's no-panic rule (CONTRIBUTING) makes an exception, explicitly per AGENTS.md §3 ("forbidden: panic outside recovered HTTP middleware boundaries").
- **AST-based signature validation by arity is enough for MVP.** Full type-identity verification at codegen would require resolving `*kit.RequestEvent`, `kit.ResolveFn`, `*kit.Response`, etc. through the user's import scope — duplicating what `go build` already does. Counting parameters and results catches the obvious "func Handle()" mistake while letting the generated adapter compile its way to the real type error if signatures drift. Two-tier check: arity at codegen (fail-fast on garbage), full type at `go build`.
- **`hookssrc` mirror is a sibling of `usersrc` and `layoutsrc`, not a child.** Three independent mirror trees under `.gen/` (`usersrc/`, `layoutsrc/`, `hookssrc/`) keep the file-name conflicts away (every mirror would otherwise compete for `.gen/usersrc/hooks/hooks_server.go` vs route-encoded paths). Keeps each mirror's package name uniform too: `hookssrc` is a fixed identifier the wire code imports by alias, with no per-route encoding.

### Self-rules

1. **Read the generated/user-authored split before assuming a generated main.go.** Codegen produces `.gen/`; user owns `cmd/<app>/main.go`. New runtime contracts surface via factory functions on the gen package (`gen.Routes()`, `gen.Hooks()`, …) and through `server.Config` fields, never via a generated `main.go`. Anything that needs to run before `ListenAndServe` (Init) goes inside the server type, not in a generated wrapper.
2. **When a hook can both succeed and short-circuit, model resolve as `(*Response, error)` and reserve a sentinel error for "handler already wrote".** `errServerRouteWrote` keeps the pipeline writer's job simple: response is non-nil → write it; response is nil and err is nil → resolve already wrote (no-op); err is non-nil and not the sentinel → route to HandleError. Three branches, no overlap.
3. **Sentinel `errors.As` checks always run before the user-supplied HandleError.** Sentinel types (RedirectErr, HTTPErr, FailErr, SafeError) carry framework-defined semantics that HandleError cannot improve on. HandleError owns the *unsorted-yet* error set; sentinels are pre-sorted.
4. **Backward-compat shims for default-only behavior live in the writer, not in hook discovery.** When the user's HandleError absent (identity default), legacy interface checks (`httpStatuser`) still apply because pre-hooks code authoring relied on them. The shim is conditional on the SafeError being the canonical default-500 pair — user-authored HandleError returning 500 deliberately is not overridden.
5. **Panic recovery at the framework boundary is the documented exception to the no-panic rule.** Recover once, around the Handle call, and convert the panic value to an error. Any deeper recovery (per-route, per-Load) is the user's responsibility — wrapping every layer doubles the cost and the diagnostic noise.
6. **Mirror trees for user `.go` files use one fixed package name per concern, not per-route encoding.** `hookssrc` is the package every gen file imports by alias. Concern-bounded mirror trees (`usersrc/`, `layoutsrc/`, `hookssrc/`) keep filename collisions impossible without forcing a path-encoding scheme on top.


## Phase 0r — page options + REST endpoint dispatch (#79 + #29) (2026-04-30)

### Insight

- **TrailingSlash policy lives on the matched route, not on the URL — but the URL drives match selection.** The chicken-and-egg flips when the policy flips: a route declared `/about` with `TrailingSlashNever` still receives requests for `/about/` (the radix tree returns a miss), and a route declared `/about` with `TrailingSlashAlways` cannot redirect away from itself without first matching. Resolution: try the match as-is; on miss, retry with the toggled trailing slash; if the retry hits, `trailingSlashRedirect` then fires the canonical 308 against the now-known route policy. The retry only runs on miss, so canonical traffic still pays a single tree lookup.
- **`Options` defaults to its zero value on legacy routes.** The router's `Route` struct carries `Options kit.PageOptions`; pre-Phase-0r manifests never populate it. The zero value sets `SSR=false` and `TrailingSlash=Default`, which would break older routes if SSR=false were treated as "skip render". The pipeline distinguishes "SSR explicitly false" from "no options resolved" by comparing the entire struct against `kit.PageOptions{}`. Cleaner than a sentinel pointer; survives `if opts == (kit.PageOptions{})` cleanly because the struct is comparable.
- **Cascade resolution belongs in codegen, not in the runtime.** SvelteKit-style page options cascade outer→inner across the layout chain. Computing the effective value at request time would force the pipeline to walk the layout chain on every match. Resolving at codegen and storing the materialized `kit.PageOptions{...}` literal in each manifest entry keeps the request path branch-free: one struct read, one comparison, decide the route.
- **Const scanner mirrors hookscan but recognizes `*ast.GenDecl` with `Tok=token.CONST`.** `go/parser` exposes top-level constants as `*ast.GenDecl` with `Tok=token.CONST`; each `*ast.ValueSpec` lists Names and Values in lockstep. Bool literals come back as `*ast.Ident{Name:"true"|"false"}`; selector-form `kit.TrailingSlashAlways` comes back as `*ast.SelectorExpr`. Bare `TrailingSlashAlways` (dot-import case) appears as `*ast.Ident`; recognizing both forms keeps the scanner honest without a full type-check.
- **Renaming `emitServerStub` to `emitRESTRoute` was the right move, not adding a parallel function.** The placeholder existed only to satisfy `route.HasServer` in earlier phases. Keeping it alongside a new dispatcher emitter would double-write the same `server.gen.go` file. Step 0 (delete dead code) cleared it before the new emitter landed; the manifest's `Server: <pkg>.Handlers` reference now resolves to the dispatcher map populated by the user's verb functions, not a TODO comment.
- **Issue body referenced `+server.go`, but ADR 0003 amendment landed `server.go` (no plus).** The brief flagged this convention conflict explicitly: the file convention was settled in Phase 0i-fix and `routescan/scan.go` already gates on `server.go`. Built against the actual scanner contract; PR description carries the convention note so anyone re-reading #29 sees the file naming corrected at codegen time.
- **`<div id="app"></div>` is the SSR=false body, not a synthesized full page.** Per #79 acceptance: "renders empty shell server-side". The framework cannot speculate on layout chrome for SSR=false routes (layouts may declare data that never gets loaded). Emitting just the mount point keeps the contract small and lets the Vite client (#34) take over on first hydration.

### Self-rules

1. **When request shape (path) and resource shape (route) disagree, retry the match before declaring 404.** A miss with a togglable suffix (trailing slash, case folding, etc.) deserves one retry under a canonicalization rule recorded on the matched resource. The retry is bounded; an unconditional second pass is fine when the first miss is cheap. Never paper over the disagreement by registering both forms in the radix tree — duplication breaks `Pattern`-keyed lookups elsewhere.
2. **Comparable struct zero values double as a "not set" sentinel for runtime feature gates.** When a value type is comparable and its zero state is meaningful in the legacy code path, `if v == (T{})` is cheaper than threading a `*T`. Document the convention at the comparison site so future readers understand "zero means legacy, not 'all defaults explicit'".
3. **Cascade resolution happens at codegen, not at request time.** Any feature that walks the layout chain (page options, layout-level cookies-namespace, layout-level prerender hints) gets resolved during `Build` and materialized into the manifest entry. The runtime reads scalars; never re-walks.
4. **AST const-decl scanning recognizes both `*ast.SelectorExpr` (qualified) and `*ast.Ident` (dot-import) forms for typed constants.** Only typed boolean and enum constants need this; integer literals or string constants are simpler. Reject unrecognized identifiers explicitly so a typo (`TrailingSlashWeird`) fails the build instead of degrading to the zero value.
5. **When renaming an old placeholder emitter to a real one, delete the placeholder first.** Step 0 of the project's structural-refactor rule. The dead function sits in the diff for one commit alongside the manifest.go updates that made it obsolete; a separate cleanup commit is overkill for a single function. Deleting in the same commit as the replacement keeps reviewers' eyes on the new emitter rather than the parallel "emitServerStub still exists" question.
6. **File convention disagreements between issue body and ADR amendment go to the ADR.** Issue #29 wrote `+server.go`; ADR 0003 (Phase 0i-fix) wrote `server.go`. The PR description carries the discrepancy note so anyone reading #29 alongside the PR sees that the convention was already settled. Never edit a closed RFC issue body to retroactively match later amendments — keep the change log in the ADR.

## Phase 0t — error boundary rendering (#28) (2026-04-30)

### Insight

- **Boundary metadata belongs on the matched route, not on the event.** `handlePipelineError` runs after the inner pipeline blew up, so it cannot re-walk the route tree to discover the +error.svelte chain. The natural fix is a small back-channel: a `*router.Route` pointer captured by a closure inside `handle()` that the resolve closure stamps on a successful match. Threading this through `respondWithError(... route *router.Route, ...)` keeps the rest of the error path untouched and avoids leaking router types into kit.
- **Outer-vs-inner is a depth count, not a slice copy.** `ErrorBoundaryLayoutDepth = layoutsAtOrAbove(chain, boundaryDir)`. The runtime composes `LayoutChain[:depth]` outer→inner around `route.Error(...)`, no allocation, no parallel "errorChain" abstraction. This reuses the layout composition pattern that Phase 0k-A landed; the only new structural piece is the `Error ErrorHandler` field.
- **A directory holding only +error.svelte still needs a generated package.** routescan already promotes any dir with a special file into a ScannedRoute, so `route.HasError && !route.HasPage && !route.HasServer` is reachable. Codegen must emit `error.gen.go` for that dir even though manifest.go skips it from the `entries` slice — the manifest's import block needs the package alias regardless. Tracking error imports separately (next to layoutImports) keeps the alias reservation logic uniform.
- **Error template binds `data` to `kit.SafeError` directly, no wrapper struct.** Issue #28's example wrote `ErrorData{Error: safe}`; the simpler shape uses kit.SafeError as the parameter type and lets users write `{data.Code}` / `{data.Message}` / `{data.ID}`. One less layer of indirection, one fewer alias to maintain in `+error.svelte`. The codegen emits `func (e ErrorPage) Render(w *render.Writer, ctx *kit.RenderCtx, data kit.SafeError) error` so the template lowering reuses the existing mustache/element pipeline unchanged.
- **Recursive boundary failure must drop back to plain text, not re-enter the boundary.** When `route.Error(...)` itself returns an error during render, calling HandleError again risks an infinite loop if the SafeError shape is what the boundary chokes on. `respondWithError` short-circuits to `writeSafeError` (the Phase 0h plain-text writer) on failure and logs the boundary error once.
- **Manifest emit conditionals must cover all boolean combinations of feature flags.** Pre-0t the import block only branched on `hasPage || hasLayout` for `kit`/`render`/`fmt`. A route table with only error packages (no page, no layout) needed `kit` and `render` but not `fmt`. The fix is a `switch` block ordering page/layout (with `fmt`), then error (without `fmt`), then options-only (kit alone), then default. Generated source stays minimal but compiles for every shape of input scan.

### Self-rules

1. **When the pipeline needs the matched route after the resolve closure errors, capture it via a pointer-to-pointer at the `handle()` scope.** The closure stamps `*matched = route` on a successful match; the error path reads it through the captured pointer. Avoid threading the route into kit.RequestEvent — that would surface a router-internal type to user code, and Locals as a stash bag is too easy to overwrite from a Handle hook.
2. **Reuse `LayoutChain[:depth]` slicing for any feature that says "render only the layouts above some boundary".** A new error-chain abstraction would duplicate the composition loop and require parallel emission in the manifest. The only metadata the runtime needs is `(handler, depth)`. Keep the chain authoritative; new boundaries are slices of it, never replacements.
3. **Codegen metadata for "covers this route" relationships propagates to every leaf during scan, not synthesized at emit time.** `ScannedRoute.ErrorBoundaryDir` / `ErrorBoundaryPackagePath` / `ErrorBoundaryLayoutDepth` is computed once during routescan when the directory tree is in hand. The manifest emitter then reads three flat fields per route — no second tree walk, no per-emission ancestor lookup. Same pattern as `LayoutPackagePaths`.
4. **For a "render this kit type directly" template, don't introduce a per-template DataType alias.** The error-page lowering binds `data kit.SafeError` because `kit.SafeError` is already exported and stable. Adding `type ErrorData = struct{...}` would force users into a synthetic field name (`{data.Error.Message}` vs `{data.Message}`) for zero gain. Reserve the alias dance for when codegen needs to splice user-declared fields into the parameter type (PageData, LayoutData).
5. **Recursive failure modes get an explicit fallback, not a panic-recover loop.** When the boundary itself fails, the response path drops to the simplest writer the framework has (plain text). Logging the boundary error once at Error level lets ops see it; a user can debug by removing the +error.svelte temporarily. Never retry an HTML render with a "simpler" boundary template — the failing template might be the only one the user owns.
6. **Manifest import emission uses a switch over feature presence, not chained `if` clauses.** As features add (pages, layouts, errors, options-only), the import block needs every combination of `kit`/`render`/`fmt` resolved correctly. A switch makes the precedence obvious at a glance: page/layout owns the broadest import set (incl. fmt for type-mismatch errors), error owns kit+render, options-only owns kit alone. Stays gofumpt-clean and reviewer-comprehensible.

## Phase 0w — codegen template blocks ({#await} + {#key}) (#57 + #58) (2026-04-30)

### Insight

- **AST + parser + dispatcher were already wired with TODO stubs.** Both `AwaitBlock` and `KeyBlock` had AST nodes, parser entry points (`parseAwaitBlock` / `parseKeyBlock`), and `emitNode` switch arms returning `// TODO`. The Phase 0w work was the lowering body plus position/diagnostic rules — same shape as Phase 0v. Always grep for the AST type before assuming greenfield: a prior phase may have left a stub waiting for the lowering to land.
- **`{#await x then v}` shorthand is a Svelte syntax our parser does NOT support.** Parser only accepts the longhand `{#await fn()}{:then v}...{:catch e}...{/await}`. The first fixture pass used the shorthand from the issue body verbatim, parsed cleanly into an AwaitBlock with `Expr="fetchUsers() then users"` and empty Then/ThenVar — codegen emitted nothing surprising but the goldens looked wrong. 1. **When a feature ships content that lives outside any existing Go module, give it its own `go.mod` rather than copying bytes into a sibling package.** `embed` cannot escape its module root; dual-source content drifts the moment one copy gets edited. 2. **Snapshot tests for content that wants to evolve assert anchors, not byte-equality.** Pick the load-bearing strings (URLs, function names, build constraints, cross-links) and require their presence. 3. **Anti-pattern coverage is a positive assertion, not a window heuristic.** "Banned token must not appear without rejection context" is brittle (the rejection heading may be one section away). Flip it: "rejected token must appear at least once" — guarantees the rejection bullet exists, runs in O(n) over each file, no false positives.

## Phase 0y — CSP nonce + sitemap/robots helpers (#66 + #91) (2026-04-30)

### Insight

- **The CSP middleware fits cleanly above Handle, not inside it.** Header writes performed before `runHandle(ev, resolve)` survive every downstream path: success `writeResponse`, error `respondWithError`, short-circuit `errServerRouteWrote`, and 404 `writeSafeError`. A user-installed Handle hook still sees the nonce on `ev.Locals` because `applyCSP` runs first; their hook can override the header by Set-ing it on the response. One source of truth, no per-path duplication.
- **Go's HTTP transport silently trims leading whitespace from header values.** The `kit.NonceAttr(ev)` helper returns ` nonce="..."` (leading space deliberate) so users embed it directly into a `<script` tag mid-attribute. The first integration test asserted the leading space round-tripped through `resp.Header.Get`; it doesn't. The wire-trimmed view is normal — the helper's contract is the in-memory string, not what survives a transport. Test the helper's return value directly OR strip the leading space before sending; do not couple the helper's documented shape to net/http's normalization rules.
- **A 16-byte / 22-character base64 nonce is the OWASP sweet spot.** `crypto/rand.Read` into a `[16]byte` plus `base64.RawURLEncoding` skips padding and produces a token safe for HTML attributes without escaping. RawStdEncoding would also work but introduces `+` and `/` which look unfriendly mid-attribute even though they're attribute-safe. Pick one encoding (RawURL) and stay consistent across the helper, the integration test, and the docs.
- **Default directives are rebuilt per request, not memoized.** `DefaultCSPDirectives()` returns a fresh map each call so `BuildCSPHeader` can mutate `script-src` to splice the nonce token without leaking state across requests. Memoizing the defaults at package scope and then `append`-ing the nonce into the cached slice is a footgun: `append` may reuse the cached backing array on the second call and produce a header containing two nonces. The "always copy on read" rule is cheaper than tracking when copy-on-write is needed.
- **`bytes.Buffer.WriteRune` is the right primitive for XML escaping.** SitemapBuilder iterates the input string with `for _, r := range s` so the loop sees runes, not bytes. Calling `buf.WriteByte(byte(r))` would silently corrupt non-ASCII codepoints; `buf.WriteRune` writes the full UTF-8 sequence. Cheap reminder: when a string-iteration loop's loop variable is `r` (rune), the writer call must match the type.
- **Hydration nonce wiring is a future-phase concern, not a same-PR change.** The hydration `<script id="__sveltego__" ...>` tag does not exist in the runtime yet — it lands with #34 (Vite client). Trying to add nonce attribution speculatively means inventing an emit point that doesn't exist, then rewriting it once #34 chooses a different shape. The right move: ship `kit.NonceAttr(ev)` for user-emitted scripts now, add the hydration tag with `nonce="..."` baked in when #34 lands, and document the deferral explicitly so reviewers don't expect both halves in this PR.
- **Builder API for robots.txt mirrors the file format, not a graph data structure.** `r.UserAgent("X").Allow("/").Disallow("/admin/")` reads top-down like the on-disk file. Encoding the same intent through a `Group` struct with `(*Group).Allow(...)` would force the user to remember which group they're holding; the chainable form keeps the current group implicit and switches it on each `UserAgent` call. The implicit `*` group when a rule is added before any UserAgent matches the spec's "rules without a User-agent attach to all crawlers" reading and saves the most common single-group case from a redundant `.UserAgent("*")`.

### Self-rules

1. **Middleware that needs to set headers on every response path runs before the path branches.** Add the side effect in `handle()` immediately after `NewRequestEvent`, never inside one of the path-specific writers. Verifying coverage means writing one integration test per terminal path (success / error / 404 / hook short-circuit) and asserting the header is present.
2. **When the user-facing helper returns a string with whitespace prefixes/suffixes, document and test the contract directly — not through net/http round-trip.** Leading-space attributes are a normal HTML idiom; net/http's header canonicalization is not the audience. Unit-test the helper's return value; integration-test the *behavior* the user gets when they paste the string into HTML output.
3. **Maps returned from "default config" functions return fresh allocations every call.** Don't cache and don't share a returned map between requests. The cost of one heap allocation per request is negligible against the cost of debugging a header that contains two nonces because `append` reused a backing array.
4. **Defer wire-up to a not-yet-existing emit point with an explicit note in the PR description.** When the helper lands but the consumer is still waiting on a future issue, both the PR and the lessons entry call out the deferral and identify the issue that will close the loop. Never stub a placeholder emit "for symmetry" when the real shape is unknown — it'll be wrong shape when the consumer lands and the stub becomes a rebase hazard.
5. **Crypto choices in framework-owned helpers go through the standard library, not third-party utilities.** `crypto/rand` + `encoding/base64` covers nonce generation in 5 lines with zero dependency surface. The instinct to reach for a "random string" helper from an internal utils package is wrong — `math/rand` finds its way into those packages eventually, and CSP nonces with predictable RNG are an XSS bypass not a security feature.

## Phase 0aa — streaming SSR with kit.Streamed (#64) (2026-04-30)

### Insight

- **A streaming render path can't reuse the buffered Response carrier.** The existing pipeline returned `*kit.Response{Body: []byte}` and let `writeResponse` fan it out at the end. Streaming flips the dependency: the render path needs the live `http.ResponseWriter` to call `Flush` between chunks. The clean fix mirrors the existing `+server.go` short-circuit — emit a sentinel error (`errStreamingWrote`) that the outer `handle()` matches with `errors.Is` and returns without further writes. Sentinels are honest about "this code already wrote the response, don't double-write" in a way a nullable Response field can't.
- **Reflection over the user's PageData is the only deferral-friendly registration mechanism.** Forcing the user to call `ev.RegisterStream(s)` puts the goroutine lifecycle on the user's plate; making `Stream(fn)` synchronous defeats the purpose. The chosen shape — `Stream(fn)` spawns the goroutine immediately, the pipeline walks the typed PageData via `reflect.Value.Field(i)` looking for `kit.StreamedAny` implementers — keeps the user surface to "return a value" while the runtime owns timeouts, cancellation, and emission order. Cost: top-level fields only (no nested traversal). That's a deliberate trade — nested streaming is a nice-to-have that rewards over-engineering before there's a user.
- **Generic types satisfy interfaces through method sets, not the type parameter.** `*Streamed[T any]` with methods `StreamID() uint64` and `WaitAny(ctx, timeout) (any, error)` satisfies the package-level `StreamedAny` interface for every T without a per-instantiation registration. The pipeline's reflection walk only needs the interface check; it never inspects T. This is the same pattern Go's stdlib uses for `sync.Map` consumers — the type-erased interface is the runtime contract, the generic struct is the user contract.
- **`json.Marshal` does not HTML-escape; `json.Encoder.SetEscapeHTML(true)` does.** A naive `<script>__sveltego__resolve(id, JSON)</script>` emission with `json.Marshal` is XSS-vulnerable when the streamed value contains `</script>`. The fix is targeted: post-process the encoded bytes to replace `</` and `<!--` with their `<` escapes. Other `<` characters can stay raw because they're inside a JSON string literal, which the JS parser treats as ordinary text. Don't reach for the encoder — its HTML mode escapes more aggressively than needed and breaks structural characters (`>` inside arrays). The narrow escape function is auditable in 8 lines.
- **Goroutine cleanup is the user's job; the runtime documents the contract.** `Stream(fn)` does not pass a `ctx` into `fn` — it's the user's closure, the user captures whatever context they need (typically the LoadCtx's request context). The pipeline's `Wait(ctx, timeout)` honors the request context for the *waiter* side, not the producer. Conflating the two would force `Stream` to grow a context parameter that 90% of users would copy-paste from a tutorial without thinking. Document the convention; let `select { case <-ctx.Done(): }` inside fn be the user's explicit opt-in.
- **Flush after every emission, not at the end of the loop.** A buffered `for _, s := range streams { emit(s); collected = append(collected, payload) }` followed by one `FlushTo(w)` defeats streaming entirely — the client sees one big write at the end. The flush must be per-iteration so each `<script>__sveltego__resolve(...)` chunk hits the wire as soon as its producer goroutine completes. Verify with a test that holds one stream blocked while another resolves and asserts byte-order at the client.

### Self-rules

1. **Sentinel errors are the right channel for "the inner handler already wrote the response".** Match `errServerRouteWrote`'s shape: a package-private `errors.New` value, an `errors.Is` check in the outer `handle()`, no further writes after the match. New paths that take ownership of the `http.ResponseWriter` (streaming, websocket upgrade in user code, etc.) should mint a new sentinel rather than overload an existing one — `errors.Is` reads as documentation when the names are distinct.
2. **Reflection-driven discovery walks top-level struct fields only.** Recursive descent into maps, slices, and pointers tempts callers into nested-stream patterns whose error semantics are undefined ("which stream's timeout fires when the parent struct is nil?"). Keep the walk shallow, document the constraint in the godoc, and let users hoist nested streams to the top level if they want them registered.
3. **Type-erased interfaces for generic types live in the same package as the generic type.** `StreamedAny` next to `Streamed[T]` keeps the contract greppable and lets the type-parameter-free interface stay an internal-ish API even though it's exported. Don't put the interface in the consumer package (the pipeline) — that's where it gets called, not where it gets defined. Same convention as `io.Reader` living in `io`, not in every reader's caller.
4. **HTML/script escapes for inline JSON neutralize "</" and "<!--" only, not all of "<".** The narrow rule is faster, easier to audit, and doesn't break structural characters. The standard quote — Go's `text/template` html.go — does the same dance. Avoid `json.Encoder.SetEscapeHTML(true)` for inline-script payloads because it mangles `>` and `&` inside arrays/strings, producing valid JSON but brittle reading inside `<script>`.
5. **`Flush` between chunks, not at the end.** `bytes.Buffer`-style accumulation with one terminal flush is the buffered render's pattern; streaming render flips it. Per-chunk flush is the load-bearing behavior — verify with a test that asserts byte-order on the client side, not just final-body content. The test must hold one stream blocked while another resolves so the assertion fails when flush is misplaced.
6. **The `ctx` for the producer goroutine is the user's closure responsibility; the runtime owns the waiter context.** `Stream(fn)` deliberately accepts `func() (T, error)` not `func(context.Context) (T, error)` — adding the context parameter would imply the runtime cancels the producer on request abort, which it can't (Go has no goroutine kill). The runtime cancels the *waiter* via `Wait(ctx, timeout)`; the producer either finishes naturally or leaks. Document the leak risk; let user code wire `<-ctx.Done()` inside fn when it matters.
7. **Streaming integration tests assert byte-order, not just byte-presence.** "the body contains both the shell and the payload" passes for buffered AND streaming responses. The assertion that distinguishes them is "the shell can be read before the payload-blocking goroutine releases" — a `bufio.Reader.Read` with a deadline before the release-channel send catches a misplaced flush that a `ioutil.ReadAll`-style test would miss.

## Phase 0z — Svelte 5 runes ($props/$state/$derived/$effect/$bindable) (#43-#47) (2026-04-30)

### Insight

- **Hand-roll the rune line classifier; do not coerce go/parser into accepting Svelte syntax.** First attempt wrapped loose top-level statements inside a synthetic `func __sveltegoTopLevelStmts__()` so go/parser would consume the rune lines as statements, then reverse-engineered the destructure pattern from the resulting AST. Working but fragile: every shape variant (type annotation, rest spread, bindable default) required a different post-parse recovery rule, and the synthetic function shape leaked into error messages. Rewrote as a line-by-line classifier that recognizes `let ` and `__sveltegoRune__effect` prefixes, joins continuation lines until paren-balance settles, and parses only the RHS through `parser.ParseExpr`. The shape of each rune is small and well-defined; matching it directly costs ~50 LOC vs ~150 LOC of post-parse archaeology.
- **`$props()` on a top-level page is a component-internal contract, not a function parameter.** Issue #43's example shows `func Render(props Props, w *render.Writer)`, but `+page.svelte` Render signatures are owned by the runtime — `(w, ctx, data PageData)`. Adding a `props Props` parameter would require touching the manifest/router/server/runtime, all of which are explicitly out of scope for this phase. The clean compromise: emit `Props` struct + `defaultProps` helper at package scope, then declare `var props Props; defaultProps(&props)` at the top of Render's body. Templates reference `props.X` as the issue intends; the parent-passes-props story lands when component nesting (#29) does, at which point the Render signature can grow without breaking existing routes.
- **Bindable is a `$props()` field with a tag, not a separate construct.** Issue #47 wants two-way binding metadata exposed to the client codegen pass (#34). Encoding bindable-ness as a struct tag (`kit:"bindable"`) keeps the metadata adjacent to the field rather than in a parallel map; client codegen reads the tag via reflect when it lands. No `kit.Bindable[T]` wrapper type, no metadata table — the tag IS the metadata. Saves a public API surface that the v0.4 milestone hasn't finalized.
- **`$effect` on the server is empty, full stop.** Tempting to emit a marker comment, a hydration anchor, or an `_ = effectSlot` placeholder so future passes have a hook. None of those have a consumer yet, and the issue explicitly says "remove from server emit". Emitting `// $effect elided on SSR` keeps the diff visible without producing runtime cost; emitting nothing would be cleaner still but loses the documentation breadcrumb when a developer is debugging "why isn't my effect firing on the server".
- **`$derived.by` IIFE shape requires explicit parens around the function literal.** `func() T { ... }()` parses but `format.Source` reformats it as `func() T { ... }()` — fine until the body has comments or the receiver carries a `func(...) T { ... }` shape that go/printer reflows differently. Wrapping in parens (`(func() T { ... })()`) makes the precedence explicit and survives every formatter rule. Three extra characters; zero edge cases.

### Self-rules

1. **When a rune call has a well-defined shape, parse the shape directly via line-classifier + `parser.ParseExpr` on the RHS — do not coerce the source into a Go-parseable shape and recover meaning post-parse.** Coercion-then-recovery requires writing two parsers (the coercer and the recoverer) and they will disagree on edge cases. A direct line classifier (`HasPrefix("let ")` + paren-balance join + `ParseExpr`) is one pass and one parser.
2. **When a feature in a downstream-owned signature is "out of scope, but the issue example shows it", emit the feature at package scope and consume it inside the existing signature.** `Props` struct + `defaultProps` at package scope, `var props Props` at the top of Render's body. Template references `props.X` as the issue intends without touching the runtime/manifest contract. Re-evaluate when the dependent issue (#29 component nesting) lands.
3. **Metadata for client-side wiring lives on the field via struct tags, not in parallel maps.** `kit:"bindable"` ships with the Props struct itself; the client codegen pass reads it via `reflect.StructTag.Get`. A `bindableFields = map[string]bool{...}` table would have to be regenerated and kept in sync; the tag is the source of truth.
4. **Loose top-level statements that are NOT recognized rune calls remain top-level declarations and fail with the existing "expected declaration" error.** The line classifier only routes lines beginning with `let ` (containing rune tokens) or `$effect`-prefixed calls. Bare assignments (`x := 1`) sit on the residue path and surface the same diagnostic as before. Don't expand the classifier to "any statement"; that's a different issue, and it needs the runtime concept of a render-body local scope to be useful.
5. **Server-side rune lowering is one-shot evaluation. SSR has no notion of reactivity, no state-tracking, no cleanup callbacks.** `$state(x)` becomes `name := x`; `$derived(expr)` becomes `name := expr`; `$effect(fn)` becomes nothing. Document this in the codegen comment block so reviewers don't ask "where does the reactivity go?" — it goes to the client bundle, which Vite handles independently.

## Phase 0bb — v0.4 element handlers (#49 + #50 + #52 + #53) (2026-04-30)

### Insight

- **Closure-based slot lowering is the same primitive as the existing layout `children` parameter — just per-name and per-prop.** Layouts already pass `children func(*render.Writer) error` as the last param to `Render`; the natural extension for components is a `Slots` struct whose fields are typed closures. No new runtime registry, no string-keyed lookup, no separate "slot system". `<slot name="header"/>` lowers to `slots.Header(w)`; `<slot item={item}/>` to `slots.Default(item, w)`. Same code path on both sides.
- **`<svelte:component this={X}>` does not need a per-site interface generator.** The issue body proposed a full adapter machinery (per-site `pageComponent` interface, generated wrapper structs per candidate). For the SSR-only lowering Phase 0bb landed, that's overkill: the user's expression must already be a value that exposes `Render(*render.Writer) error`, so codegen just emits `expr.Render(w)`. Any "static set" optimization (switching on a discriminator) is the user's switch in their own Go code — codegen takes the result. Less generated code, no synthetic interface names, and the receiver shape stays identical to `Component.Render(w)` already used for layouts.
- **Position validation works through a single `Builder.nestDepth` counter, not a separate AST walk.** `<svelte:body|window|document>` must appear at the template root. Adding a depth counter to `Builder` that increments inside `<element>`, `{#if}`, `{#each}`, `{#await}`, `{#key}` makes the validation a one-line check at emit time (`if b.nestDepth > 0 { fail }`). A pre-walk pass would have duplicated the recursion structure already in `emitChildren`. The defer-based decrement keeps the increment/decrement pairs visually adjacent so future `{#snippet}` and similar nestable constructs can join the count without rewiring control flow.
- **`AttrLet` is a parser-level kind, not a codegen heuristic.** Initially considered detecting `let:foo` by string-prefix in codegen alongside the existing classifyAttribute switch. That would have placed Svelte syntax knowledge in two layers. Adding `AttrLet` to the AST and routing `let:` through `classifyAttribute` keeps the parser the single source of attribute classification truth — codegen just consumes a typed `Kind`.
- **Slot-prop closure args type as `any` until the call site forces concrete types through Go's inference.** Component templates don't declare types; the producer's expression carries a Go type, the consumer's `let:foo` references it. Typing the closure as `func(item any, w *render.Writer)` lets the user cast or assert at the receiver site without forcing codegen to re-discover types from a separate Go AST pass. The cost is a missing compile-time error for `<List let:item>{item.NotAField}</List>` — `any` accepts anything; the user gets a runtime nil-pointer instead of a compile error. Acceptable trade because typed slot props would force a parallel Go type extraction pass tied to a non-existent file convention. Revisit when component imports formalize (#10).
- **The `liftSlotChildren` algorithm is intentionally one level deep.** Caller-side slot lifting (`<Card><h1 slot="header">...</h1>body</Card>` → two closures) walks immediate children only. Children of children carrying `slot=` are silently kept inside the parent slot — Svelte semantics don't bubble through nested elements. Tested by checking that `<Card><div><h1 slot="header"/></div></Card>` does NOT produce a Header slot binding (the `slot=` attribute is on a non-immediate child). The shallow walk is the right call: deep walking would conflate scope and cause silent miscompilation.
- **`<slot {item}/>` shorthand requires lexer-level mustache-in-tag support that we do not have.** First fixture pass used Svelte's shorthand syntax; the lexer rejects `{` in tag-attr position. Two paths: extend the lexer (large blast radius) or document longhand-only (`<slot item={item}/>`) for this PR. Chose longhand — the issue body's example uses both forms but the longhand is unambiguous. Filed nothing; shorthand support belongs to a future lexer extension issue if it surfaces.
- **`emitComponentCall` defers full props lifting to #10 by intent.** Phase 0bb owns slots; #10 owns component nesting + import resolution. The current emit assumes the user imports the component package under the lowercase-first-char alias (`Card` → `card`), constructs `card.CardProps{}` empty, and invokes `card.Card{}.Render(w, ctx, card.CardProps{}, _slots)`. That signature is the contract `GenerateComponent` produces. When #10 lands, the caller-side dispatch grows props composition; the slot-lifting half stays as-is.
- **Component fixtures must live OUTSIDE `testdata/codegen/` because `TestFixtures` walks that tree.** First placed component fixtures under `testdata/codegen/component/`; `TestFixtures` (in the FORBIDDEN `codegen_test.go`) tried to render them as page templates and failed on missing goldens. Moved to `testdata/component/` — same precedent as `testdata/golden/layout/` for layout fixtures. Treat `testdata/codegen/` as page-fixture-only; sibling subtrees own other generators.

### Self-rules

1. **When a new lowering shape mirrors an existing one (slots ↔ children closure), reuse the existing primitive instead of inventing parallel infrastructure.** A second runtime registry "for slots only" is a code smell — both compile-time decisions land on closure dispatch already. Same primitive = same audit surface = same bug class.
2. **Position validation that needs to know "am I at template root?" goes on `Builder` as a counter, not as a separate walk.** Increment on entering an enclosing construct, decrement on leaving (defer keeps the pair adjacent). One-line check at the validation site. Resists drift: any new nestable construct (`{#snippet}` next, then `{#await}` already covered) must remember to bump the counter to participate in root-only validation.
3. **Lowering for `<svelte:component this={X}>` calls `X.Render(w)` directly. Do not synthesize per-site interfaces.** The user's value already implements the receiver method; codegen's job is dispatch, not type wrapping. Reserve adapter generation for the day a real polymorphism case surfaces — until then, the indirection costs lines without buying anything.
4. **New attribute syntax extensions (`let:`, future `bind:` improvements) get an `AttrKind` value in the AST, not a string-prefix detection in codegen.** The parser is the single classification authority; codegen consumes typed kinds. Two-layer detection drifts the moment one layer needs to update.
5. **Receiver-side slot prop closures take `any` parameters; type recovery is the user's `let:` binding's job.** Codegen does not re-derive types from a separate Go pass for closure params. The runtime contract (positional args, last-arg `w`) is what makes the call site stable — type fidelity is a v1.0 perfectionist concern (#10 + future typed-props issue), not a Phase 0bb scope.
6. **When the issue body shows both shorthand and longhand forms of a Svelte syntax, test the longhand first against the existing lexer.** The shorthand may not parse. Confirmed earlier in Phase 0w with `{#await x then v}` shorthand; reconfirmed here with `<slot {item}/>` shorthand. The test corpus is the syntax oracle, not the issue body.

## Phase 0cc — snippets ({#snippet}/{@render}) + env codegen guard (#48 + #82) (2026-04-30)

### Insight

- **Recursive snippets need `var name func(...); name = func(...) { ... }`, not `name := func(...) { ... }`.** Go's short-var-decl form does not bind `name` inside the RHS function literal — a snippet referencing itself by name in its body would fail to compile. The split var-decl-then-assign pattern lets the closure see its own identifier through the captured outer `name` slot. Trailing `_ = name` keeps the unused-variable check quiet for snippets that are never invoked in the same scope (the future component-prop pass will read them via the parent's emitted Props struct).
- **`{@render expr}` writer goes LAST, not first.** The earlier stub emitted `p.snippet_card(w, ctx, post)` with `w` first, mimicking a method receiver convention. The Phase 0cc rewrite drops the `p.snippet_*` indirection entirely and lowers `{@render card(post)}` to `card(post, w)` — the writer threaded as the trailing argument so the call site reads like a normal Go function call. This matches the issue body's lowered form exactly and lets snippets passed as struct fields satisfy `func(T, *render.Writer) error` without reshuffling.
- **Env codegen guard hooks into `validateExpr`, not a parallel pass.** Every template expression already routes through `validateExpr` for parse-time validation. Re-parsing the source inside a separate guard would double the work and fork the diagnostic surface. Capturing the parsed `*goast.Expr` from `parser.ParseExpr` and walking it once with `goast.Inspect` for `env.StaticPrivate` / `env.DynamicPrivate` adds the leak check at the same depth as the parse error — same line:col, same `*CodegenError` shape.
- **`go/parser.ParseFile` against a synthetic `func f(<params>) {}` is the right validator for snippet param lists.** Trying to write a hand-rolled comma splitter for `name Type, name2 Type2` underestimates Go's grammar (function types in params, generic instantiation, variadic, named returns shaped like params). Wrap params in a synthetic file and let go/parser do the lifting; the failure mode is a `*CodegenError` with a clear "invalid {#snippet} parameter list" prefix and the upstream go/parser message preserved for context.
- **Test driver's `assertErrorGolden` requires the error file to PRE-EXIST before `GOLDEN_UPDATE=1` will write to it.** The walker's `os.Stat(errGoldenPath)` decides whether the fixture is on the error path. Without a placeholder file, even `-update` runs treat the case as a positive fixture and fail to materialize the error golden. Workflow: touch a placeholder, run `GOLDEN_UPDATE=1`, the placeholder is overwritten with the truncated message. Document this on every codegen-error fixture batch — easy to forget across phases.
- **Build-time substitution (#82's "resolver" half) belongs to the Vite-blocked follow-up, not this PR.** The acceptance criterion that landed here is "build fails when a +page.svelte references private env" — pure codegen guard, no resolver, no client manifest. The remaining points (static-public inlined into client, dynamic-private read at runtime never on client, missing static var fails build) all depend on a client emit pass that doesn't exist until #34 (Vite). Filed the deferral in the PR body so reviewers don't expect both halves; #117 already exists as the resolver follow-up.

### Self-rules

1. **Closure self-reference requires `var f func(...); f = func(...) { ... }`, never `f := func(...) { ... }`.** Short-var-decl is a tempting one-liner but the RHS does not see `f` until after the declaration completes, so any recursive call inside the body fails to compile. Apply this rule to every codegen lowering that produces a closure addressable by name (snippets, future named partials, future kit.Streamed handlers).
2. **Implicit framework parameters go LAST in lowered call signatures.** Putting the writer (or any future framework-injected param) at the tail keeps the user-facing call shape readable: `card(post, w)` reads like a normal call, `p.snippet_card(w, ctx, post)` does not. Same rule for snippet-as-prop typing — the field type matches what the user would write by hand (`func(Post, *render.Writer) error`), the runtime appends its writer arg.
3. **Diagnostics that gate "expression flows to client" hook into `validateExpr` rather than a separate walker.** Single parse, single AST walk, one `*CodegenError` shape, one line:col. Adding a parallel pass forks the diagnostic surface and doubles the parse cost. Reuse `parser.ParseExpr`'s output via `goast.Inspect` whenever a "is this expression safe in template context" check needs to ride alongside parse validation.
4. **Validate Go-shaped sub-grammars by wrapping them in a synthetic Go file and feeding the result to `parser.ParseFile`.** Hand-rolling a parser for "function parameter list" or "import block" or "type list" loses too many edge cases (generics, variadic, embedded selectors). The wrap-and-parse pattern leans on the language spec and is the same trick `validateStmt` already uses for `{@const stmt}`.
5. **Codegen-error fixtures need a placeholder error golden checked in BEFORE `GOLDEN_UPDATE=1`.** The fixture driver routes by file existence, not by error presence. Always: write the `.svelte`, write a placeholder `.error.golden`, run update. Skipping step two silently routes the case to the positive-fixture path and the test fails with "generate: <message>" instead of materializing the golden.
6. **When a multi-bullet acceptance list mixes "guard" and "build-time resolver", land the guard now and defer the resolver to its dependency-blocked follow-up.** #82 is two features stitched into one issue body; the guard ships independently because it has no dependency on Vite. Splitting the work at the natural seam (codegen-time rejection vs build-time substitution) lets the safety property land immediately while the inlining work waits for #34. Document the split in the PR body and lessons entry so the deferral is visible.

1. **A new emitter that consumes scan output and produces a separate gen artifact gets its own file.** Don't grow `manifest.go` beyond the route table. Reuse only the cross-cutting primitives (`Builder`, `quoteGo`); fork everything that's emitter-specific. Audit surface stays small per file.

## Phase 0dd — scoped CSS hash + svelte:options (#54 + #90) (2026-04-30)

### Insight

- **Upstream Svelte's CSS hash is DJB2-XOR over UTF-16 code units, base36, processed in reverse.** Found in `packages/svelte/src/utils.js` at `function hash(str)`: strip `\r`, init `5381`, fold from end with `((h << 5) - h) ^ charCodeAt(i)`, finalize via `>>> 0`, emit `toString(36)`. Default `cssHash` wraps as `svelte-${hash(filename === '(unknown)' ? css : filename ?? css)}`. The output length is NOT fixed at 5 chars — base36 of a 32-bit value is 1–7 chars depending on magnitude. The issue body's "5-char base36" was a heuristic, not a contract; matching upstream byte-for-byte is what matters. Verified with Node-driven golden vectors hardcoded into the Go test (avoids JS toolchain dep in CI).
- **JS signed-int arithmetic and Go uint32 modular arithmetic produce identical bit patterns for this hash.** `((h << 5) - h)` overflows to negative in JS, XOR with `charCodeAt` runs on 32-bit ints, the final `>>> 0` reinterprets as unsigned. Running the entire fold in `uint32` reproduces every intermediate bit because addition/subtraction/shift are modular under both signed and unsigned representations — only the sign bit's interpretation changes, and the XOR cleans that. Bonus: avoids `int32 -> uint32` conversions that gosec flags as G115. Lesson: when a port involves signed↔unsigned conversion at the boundary, run the whole fold in unsigned; the bits agree.
- **CSS scope class application is an AST mutation, not an emit-time concern, when element.go is FORBIDDEN.** The natural place to intercept "every element should get class=svelte-xxxxx" is `emitOpenTag` in element.go — but that file is owned by other parallel phases. Mutating Element.Attributes in a pre-emit pass (`applyScopeClass(frag, hash)`) keeps the entire feature in one new file (style.go) and uses the existing emit pipeline unchanged. The mutation is idempotent (reuses static class slot if present, wraps dynamic into interpolated). Cleaner than threading a `scopeClass string` through Builder and reading it from element.go.
- **Removing nodes from the fragment is the right way to "consume" them in extraction passes.** `<svelte:options>` and `<style>` both need to disappear from the fragment so `emitChildren` does not try to render them. Filtering in-place (`kept := frag.Children[:0]`) followed by clearing the trailing slots avoids creating a parallel "skip these" set. Same pattern used by other languages' AST passes. Pairs with the codegen invariant that a node either lowers to source or gets extracted before lowering — never both.
- **`<svelte:options>` is parsed as a regular `*ast.Element` with `Name="svelte:options"`. The "specialness" lives in codegen, not the AST.** First considered adding an `*ast.SvelteOptions` node so the parser would surface it as a typed top-level. Two reasons against: (1) the parser already emits Element for `<svelte:*>` namespace prefixes and downstream codegen handles each by name match, and (2) parser is FORBIDDEN this phase. Treating it as Element + name match keeps codegen the validation authority and matches how `<svelte:body|window|document|head|component>` are already handled. AST node types are reserved for shapes that need different tree topology, not for special semantics.
- **Boolean `<svelte:options>` attrs need pointer fields to distinguish "absent" from "explicitly false".** A `bool` field with default `false` cannot tell `<svelte:options runes={false}/>` (opt-out, must validate) apart from `<svelte:options/>` (no opinion, runes are project default). Pointer-to-bool with three states (nil/&true/&false) is the minimum. Using tristate enums would be heavier; `*bool` is conventional Go for this pattern.
- **Acceptance items deferred when their implementation site is FORBIDDEN.** `namespace="svg"` requires element.go changes (skip void-element rules, emit `<path/>`-style self-closing). element.go is FORBIDDEN this phase, so the option is captured + plumbed but does not yet alter emission. Filed as follow-up. Same for selector-target matching in #54 — captured and parameterized as MVP "scope-all-elements" simplification; follow-up issue owns the selector matching algorithm. Phased delivery beats touching FORBIDDEN files.

### Self-rules

1. **When porting a hash function from JS to Go, run the entire fold in `uint32`, not `int32`.** JS signed/unsigned dance is finalized by `>>> 0`; uint32 mod-2^32 produces the same bit pattern at every step and avoids gosec G115 conversion warnings. Verify with Node-driven golden vectors hardcoded into the test — never call out to Node from CI.
2. **Pin the upstream version in a doc comment at the function site.** "Snapshot pinned 2026-04-30, file `packages/svelte/src/utils.js`." When upgrading, the next dev re-checks the function. No external pin file needed.
3. **When the implementing site is FORBIDDEN, feature surface = AST mutation pass before emit.** Scoped-class application mutates Element.Attributes in a pre-emit walk. Same trick works for any "tag every element with X" requirement. Element-level emit code is unchanged; the new behavior lives in the new file.
4. **Boolean compile-time options that can be explicitly opt-in OR opt-out get pointer fields.** `*bool` distinguishes nil/false/true. Avoids the three-way confusion of "not set" vs "set to false".
5. **`<svelte:*>` elements live as `*ast.Element` with name match in codegen — do not invent typed AST nodes for each.** Element + name match is the existing convention (`svelte:body`, `svelte:window`, `svelte:component`). New typed nodes are reserved for cases where tree topology differs.
6. **In-place fragment filtering is the right "consume this node" pattern.** `kept := children[:0]` + clear-trailing pointers + `frag.Children = kept`. No parallel "extracted" set, no skip-flag on the iterator.
7. **Defer acceptance items whose implementation requires touching a FORBIDDEN file. File a follow-up issue and document the deferral in the PR body.** Better to ship the MVP with explicit gaps than to either (a) violate the file boundary and break parallel agents or (b) hold the entire PR until the FORBIDDEN file frees up.

## Phase 0nn — static asset precompression (#41) (2026-04-30)

### Insight

- **Don't compress at request time; serve precompressed siblings.** The issue spec calls for `.br`/`.gz` produced at build, the runtime's job is content negotiation only. That decouples the hot path from any compression library, keeps the runtime dependency-free (no `andybalholm/brotli` in `go.mod`), and lets Caddy/Nginx-style speed fall out naturally. The handler's responsibility is exactly: pick the variant, set the encoding header, delegate I/O to `http.ServeContent`.
- **`net/http`'s default transport silently rewrites compressed responses on the client side.** When a request goes out without a manually-set `Accept-Encoding`, the transport injects `Accept-Encoding: gzip`, and on the response it transparently decompresses + strips `Content-Encoding`. This guts gzip-vs-raw test assertions — the body becomes empty (decompression of fake bytes fails) and the encoding header disappears. Tests that exercise compression *must* use a transport with `DisableCompression: true`.
- **`path.Clean` collapses `..` silently — it cannot be the only traversal defense.** `path.Clean("/foo/../../bar")` returns `/bar`, not an error. By the time the handler joins with the static root, the original intent is gone. The fix is a pre-clean rejection: scan the URL path for any literal `..` segment and 400 it before cleaning. Treat traversal attempts as bad requests, not silent rewrites.
- **ETags must vary by encoding or shared caches poison.** A single ETag for `app.js` (raw) and `app.js.gz` (gzip-encoded) lets a forward proxy hand the gzip body to a gzip-only client. The validator has to encode the encoding choice. Adding `-gzip` / `-br` to the mtime+size tag is one line and closes the cache-poisoning hole.
- **Precompressed siblings need explicit `Content-Type` because `net/http` sniffs from the file head.** The first 512 bytes of `app.js.gz` look like gzip, not JavaScript, so `http.ServeContent` would emit `application/x-gzip` without an override. Deriving `Content-Type` from the *raw* file extension keeps `application/javascript` stable across all three variants — same trick as nginx's `gzip_static`.
- **Don't wire pipeline integration when the integration file is owned by another concurrent agent.** Phase 0hh owns `pipeline.go`. Solution: ship `StaticHandler` as a standalone `http.Handler` users mount themselves (`mux.Handle("/static/", http.StripPrefix(...))`) and document the wiring pattern in godoc. Phase 0hh integrates later by calling the same exported function. Single source of truth, zero merge conflict.
- **Configuration errors surface at first request, not at construction.** `StaticHandler` with `Dir == ""` returns an `http.Handler` that 500s every request rather than panicking or returning `(http.Handler, error)`. The error-returning shape would force every caller to check; the 500-on-request shape lets composition stay one-line and surfaces failure on the first hit (where logs are watched).

### Self-rules

1. **Static asset handlers serve precompressed siblings; they do not compress on demand.** Build step writes `.br`/`.gz`; runtime negotiates and reads bytes. No `compress/gzip` writer in the request path, no brotli library dependency in the runtime module.
2. **Tests that assert `Content-Encoding` or compare compressed bodies must use `&http.Client{Transport: &http.Transport{DisableCompression: true}}`.** Bake into a `rawClient()` helper in any new compression-related test file so it never gets forgotten.
3. **Reject `..` segments before `path.Clean`.** Walk the URL path's `/`-separated tokens and 400 the request if any token equals `..`. `path.Clean` is for safe shape-normalization (collapsing `./`, double slashes); it is not a security boundary.
4. **ETags for negotiated content include the chosen encoding token.** Otherwise shared caches will hand the wrong-encoding body to the wrong client.
5. **Override `Content-Type` from the raw extension when serving compressed siblings.** `http.ServeContent` sniffs the first 512 bytes; for `.gz` / `.br` siblings that gives the wrong type. Set the header before calling `ServeContent`.
6. **When a parallel agent owns the integration site, ship the new capability as a standalone http.Handler with godoc'd mount pattern.** The integrating agent calls the exported function later — no upstream merge conflict, no follow-up debt.
7. **Configuration errors in handler constructors return a 500-on-request handler, not a panic and not a `(handler, error)` pair.** Composition stays one-line; failure surfaces on the first request like any other deploy misconfig.

- **`role` attribute validation skips dynamic and interpolated values by design.** The check matches against a static role token list; a dynamic expression (`role={Dynamic}`) is by definition unknowable at codegen time. Refusing to flag is the right call — the alternative is "validate at runtime", which is out of the codegen pass's mandate.
- **Input-label association is over-permissive on purpose: `id` alone satisfies the rule.** Codegen has no view of sibling `<label for="...">` elements — they may live in a parent layout or be wired via a prop. Treating `id` presence as "labeled" matches Svelte's a11y plugin's behavior and avoids false positives at the cost of not catching `<input id="x">` with no matching `<label for="x">` anywhere. The trade matches the project's "warnings are advisory" policy.

### Self-rules

1. **Before introducing a new exported type in a package with existing diagnostic infrastructure, grep the package for `Diagnostic`/`Severity`/`Error` decls.** Reuse the existing surface unless extension is impossible without it. Parallel types fork the audit surface and break in collision-prone places (forbidden files, golden tests, lint rules).
2. **A new generator's fixtures live OUTSIDE `testdata/codegen/`.** Use `testdata/<kind>/` (e.g. `testdata/a11y/`, `testdata/component/`). The `codegen/` tree is owned by `TestFixtures`; never extend that walker's skip list unless you own `codegen_test.go` this phase.
3. **Accessible-name checks (`<a>`, `<button>`, label) recurse through block constructs, not only direct children.** Walk `{#if}`/`{#each}`/`{#await}`/`{#key}` branches the same way the codegen emit pass does. Naive immediate-child walks produce false positives on every real template.
4. **Validation rules over static taxonomies (ARIA roles, HTML elements, etc.) skip dynamic and interpolated attribute values.** A codegen-time check cannot know runtime values; flagging them produces noise. Static-only validation is the intentional shape, not a bug.
5. **`a11y` warnings are non-fatal by default.** Do NOT wire into `b.Fail`. Return `[]Diagnostic` for the caller (CLI/build) to render; promotion to fatal lives behind an explicit flag (deferred follow-up). The same standalone-callable shape lets editor integrations (`packages/lsp/`) consume the same pass.
