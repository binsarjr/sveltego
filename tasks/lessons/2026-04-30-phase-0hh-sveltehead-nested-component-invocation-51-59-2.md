## Phase 0hh — <svelte:head> + nested component invocation (#51 + #59) (2026-04-30)

### Insight

- **`<svelte:head>` lowers to a sibling Head method on the same receiver, not a separate codegen pass.** Page and Layout already declare a Render method bound to typed Data; the cleanest extension is a parallel Head method with the identical signature. The render-time pipeline allocates one side `*render.Writer`, walks the layout chain's Head outer→inner, then the page's Head, and splices the buffer between `shellHead` and `shellMid`. No new codegen entry point, no new runtime contract — the manifest grows two adapter shapes (head__<alias> + head__layout__<alias>) that mirror the existing render adapters byte-for-byte aside from the method name.
- **Head extraction belongs at codegen time before emitChildren walks the body.** A pre-pass `extractHeadChildren(frag.Children)` returns `(headChildren, bodyChildren)`; the Render path uses bodyChildren and is otherwise unchanged. Going the other way (emit-time skip of `<svelte:head>` inside emitElement) would force every nested-block emitter to re-discover the rule. The pre-pass is one slice walk and surfaces `rejectNestedHead` for the position-rule diagnostic.
- **Title dedupe is a buffer-level regex pass, not an HTML parser.** Issue #51's spec says "if multiple `<title>` tags exist, keep only the LAST". A full HTML5 tokenizer is overkill for one tag. A case-insensitive scan that locates `<title...>...</title>` spans and rewrites the buffer to drop all but the last one is ~30 lines and zero deps. The scan tolerates `<TITLE>` / `<Title>` / `<title attr="x">` and rejects `<titlebar>` via a delimiter check on the byte after `<title`. Anything that needs full HTML semantics belongs in the user's app, not in dedupe.
- **The shell's `%sveltego.head%` placeholder defines the splice point.** parseShell already split the user's app.html into `shellHead` (everything before `%sveltego.head%`) and `shellMid` (between head and body markers). Head buffer goes between them, full stop. No new shell field needed.
- **Manifest's optional adapter emission lives in maps keyed by package path, not in slice flags on entries.** `PageHeads map[string]bool` and `LayoutHeads map[string]bool` carried through ManifestOptions are constant-time membership checks during emit and require zero changes to the existing adapter slice plumbing. Threading a `hasHead bool` field on `entry`/`layoutImport` was the alternative — rejected because it forces every consumer of those structs to think about head when only a fraction of routes care.
- **Dot-namespaced component invocation `<pkg.Comp/>` is a SECOND component path alongside `<MyComponent/>`, not a replacement.** Phase 0bb's `emitComponentCall` (in slot.go, READ ONLY for this phase) handles capitalized non-dot tags with slot lifting but emits empty `MyButtonProps{}`. Dot-form needs proper Props mapping AND a fix for the `componentPackageAlias` corner (lowercase-first-char of "button.Comp" returns "button.Comp" — a syntactic mess). Adding `emitNestedComponent` in component.go for the dot-form keeps Phase 0bb's contract intact and gives #59 the full `pkg.Comp{}.Render(w, ctx, pkg.CompProps{Field: expr}, _slots)` lowering.
- **Component-prop attribute lowering reuses the existing `validateExpr` for dynamic values.** Static attrs lower to Go string literals via `quoteGo`; dynamic attrs run through `validateExpr` (the same gate template mustaches use) so unparseable expressions surface a `*CodegenError` with the same line:col + message shape users already see for other expression sites. Spread `{...rest}` is deferred to a follow-up — the issue body covers it but full reflection-driven spread needs a runtime helper that doesn't exist yet.
- **PascalCase identifier conversion for kebab-case attributes is a one-pass walk on a single byte slice.** `aria-label` → `AriaLabel`, `data_id` → `DataId`. Hyphens and underscores act as word separators; the first byte of every segment uppercases. Same shape as `slotFieldName` in slot.go but lives in component.go because the rule is about Go identifier rules, not slot conventions.
- **Child .svelte → child Go package generation is deferred to a follow-up.** The issue brief explicitly allows defering this: "If full file-system import resolution is too risky, defer with explicit follow-up issue + working subset." For Phase 0hh, the user manually creates the child Go package matching `GenerateComponent`'s contract (`Comp`, `CompProps`, `CompSlots`, `func (Comp) Render(w, ctx, CompProps, CompSlots) error`). The codegen-side caller lowering is complete; the file-system child generation is a separable concern with its own scope (cycle detection, filename normalization, on-disk layout under `.gen/components/`). Tracked as #144.

### Self-rules

1. **A side-channel emit (Head, future stream-resolved fragments, future a11y warnings) gets a sibling method on the existing receiver rather than a parallel codegen call.** Page and Layout already own their data type and component package; adding a Head method costs a manifest adapter and a routing slot on Route. A separate codegen entry point would duplicate package emission, import resolution, and prop scoping. The receiver is the unit of dispatch — keep it.
2. **Stripping a node kind from the body emit list goes through a pre-pass that returns `(stripped, residual)`, not a flag threaded through `emitNode`.** `extractHeadChildren` returns the head subtree and the rest; emit code for the body sees the same shape it always did. No new runtime branching, no `if isHead { skip }` lurking in every emitter. Use this pattern when a future special form (e.g. `<svelte:options>`) needs the same lift.
3. **Buffer-level transforms on emitted HTML (title dedupe, future CSP nonce rewriting, future asset URL rewrites) use byte-level scans against case-insensitive snapshots, not real HTML parsing.** Real parsing pulls in dependencies and broadens the failure surface to "your malformed input crashed the framework"; byte scans degrade gracefully on weird input. Document the boundary cases (case folding, attribute whitespace, malformed close tags) in a local helper godoc so future readers can audit one function instead of a 1000-line tokenizer.
4. **Optional manifest features pass through `ManifestOptions` as keyed maps, not as flags on the `entry` struct.** `PageHeads map[string]bool` is a `len(0)` no-op when the feature is unused; `entry.HasHead` would force every consumer to thread a value they don't care about. Reserve `entry` fields for the irreducible scan output.
5. **A second component-invocation path is fine when the existing path was scoped narrower.** Phase 0bb's emitComponentCall handles capitalized-only tags WITH slots. Dot-namespaced tags AND attribute-only invocations both want richer Props mapping; bolting that onto emitComponentCall would have grown that function past readable size and forced edits to a READ-ONLY file. New site, new emitter, same template-walking primitives. Dispatching by name shape (`strings.Contains(name, ".")`) in element.go is one line and keeps both paths visible.
6. **Defer file-system codegen (child component generation) when the lowering can be exercised without it.** The codegen-side caller lowering has independent value: users with hand-written child packages can ship today, and the child-generation pass becomes a pure file-emit feature that doesn't touch the lowering pipeline. File a follow-up issue with the deferred bullet list rather than stubbing half-implementations.
7. **PascalCase identifier conversion is a re-usable utility, not a per-feature helper.** `pascalIdent` for component props mirrors `slotFieldName`'s structure; consolidating them is tempting but premature — slot rules might diverge from prop rules later (slot names allow more punctuation in Svelte's spec). Keep both for now; refactor when a third site needs the same conversion.
