## Phase 0cc — snippets ({#snippet}/{@render}) + env codegen guard (#48 + #82) (2026-04-30)

### Insight

- **Recursive snippets need `var name func(...); name = func(...) { ... }`, not `name := func(...) { ... }`.** Go's short-var-decl form does not bind `name` inside the RHS function literal — a snippet referencing itself by name in its body would fail to compile. The split var-decl-then-assign pattern lets the closure see its own identifier through the captured outer `name` slot. Trailing `_ = name` keeps the unused-variable check quiet for snippets that are never invoked in the same scope (the future component-prop pass will read them via the parent's emitted Props struct).
- **`{@render expr}` writer goes LAST, not first.** The earlier stub emitted `p.snippet_card(w, ctx, post)` with `w` first, mimicking a method receiver convention. The Phase 0cc rewrite drops the `p.snippet_*` indirection entirely and lowers `{@render card(post)}` to `card(post, w)` — the writer threaded as the trailing argument so the call site reads like a normal Go function call. This matches the issue body's lowered form exactly and lets snippets passed as struct fields satisfy `func(T, *render.Writer) error` without reshuffling.
- **Env codegen guard hooks into `validateExpr`, not a parallel pass.** Every template expression already routes through `validateExpr` for parse-time validation. Re-parsing the source inside a separate guard would double the work and fork the diagnostic surface. Capturing the parsed `*goast.Expr` from `parser.ParseExpr` and walking it once with `goast.Inspect` for `env.StaticPrivate` / `env.DynamicPrivate` adds the leak check at the same depth as the parse error — same line:col, same `*CodegenError` shape.
- **`go/parser.ParseFile` against a synthetic `func f(<params>) {}` is the right validator for snippet param lists.** Trying to write a hand-rolled comma splitter for `name Type, name2 Type2` underestimates Go's grammar (function types in params, generic instantiation, variadic, named returns shaped like params). Wrap params in a synthetic file and let go/parser do the lifting; the failure mode is a `*CodegenError` with a clear "invalid {#snippet} parameter list" prefix and the upstream go/parser message preserved for context.
- **Test driver's `assertErrorGolden` requires the error file to PRE-EXIST before `GOLDEN_UPDATE=1` will write to it.** The walker's `os.Stat(errGoldenPath)` decides whether the fixture is on the error path. Without a placeholder file, even `-update` runs treat the case as a positive fixture and fail to materialize the error golden. Workflow: touch a placeholder, run `GOLDEN_UPDATE=1`, the placeholder is overwritten with the truncated message. Document this on every codegen-error fixture batch — easy to forget across phases.
- **Build-time substitution (#82's "resolver" half) belongs to the Vite-blocked follow-up, not this PR.** The acceptance criterion that landed here is "build fails when a +page.svelte references private env" — pure codegen guard, no resolver, no client manifest. The remaining points (static-public inlined into client, dynamic-private read at runtime never on client, missing static var fails build) all depend on a client emit pass that doesn't exist until #34 (Vite). Filed the deferral in the PR body so reviewers don't expect both halves; #117 already exists as the resolver follow-up.

### Self-rules

1. **Closure self-reference requires `var f func(...); f = func(...) { ... }`, never `f := func(...) { ... }`.** Short-var-decl is a tempting one-liner but the RHS does not see `f` until after the declaration completes, so any recursive call inside the body fails to compile. Apply this rule to every codegen lowering that produces a closure addressable by name (snippets, future named partials, future kit.Streamed handlers).
2. **Implicit framework parameters go LAST in lowered call signatures.** Putting the writer (or any future framework-injected param) at the tail keeps the user-facing call shape readable: `card(post, w)` reads like a normal call, `p.snippet_card(w, ctx, post)` does not. Same rule for snippet-as-prop typing — the field type matches what the user would write by hand (`func(Post, *render.Writer) error`), the runtime appends its writer arg.
3. **Diagnostics that gate "expression flows to client" hook into `validateExpr` rather than a separate walker.** Single parse, single AST walk, one `*CodegenError` shape, one line:col. Adding a parallel pass forks the diagnostic surface and doubles the parse cost. Reuse `parser.ParseExpr`'s output via `goast.Inspect` whenever a "is this expression safe in template context" check needs to ride alongside parse validation.
4. **Validate Go-shaped sub-grammars by wrapping them in a synthetic Go file and feeding the result to `parser.ParseFile`.** Hand-rolling a parser for "function parameter list" or "import block" or "type list" loses too many edge cases (generics, variadic, embedded selectors). The wrap-and-parse pattern leans on the language spec and is the same trick `validateStmt` already uses for `{@const stmt}`.
5. **Codegen-error fixtures need a placeholder error golden checked in BEFORE `GOLDEN_UPDATE=1`.** The fixture driver routes by file existence, not by error presence. Always: write the `.svelte`, write a placeholder `.error.golden`, run update. Skipping step two silently routes the case to the positive-fixture path and the test fails with "generate: <message>" instead of materializing the golden.
6. **When a multi-bullet acceptance list mixes "guard" and "build-time resolver", land the guard now and defer the resolver to its dependency-blocked follow-up.** #82 is two features stitched into one issue body; the guard ships independently because it has no dependency on Vite. Splitting the work at the natural seam (codegen-time rejection vs build-time substitution) lets the safety property land immediately while the inlining work waits for #34. Document the split in the PR body and lessons entry so the deferral is visible.
