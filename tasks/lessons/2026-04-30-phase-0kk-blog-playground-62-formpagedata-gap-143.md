## 2026-04-30 — Phase 0kk: blog playground (#62) + Form/PageData gap (#143)

### Insight

- The blog playground (`playgrounds/blog/`) wires markdown rendering, pagination, the `[slug]` dynamic param, layout chrome, `ev.BindForm` + `kit.ActionMap` + `kit.ActionRedirect`, and `kit.Cookies`. Index works end-to-end; post-detail returns HTTP 500 because of a real codegen contract gap surfaced for the first time in this app.
- Codegen unconditionally appends `Form any` to PageData when `HasActions=true` (`internal/codegen/codegen.go`). The render adapter strict-casts `data.(PageData)` (`internal/codegen/manifest.go`). A user `Load()` that returns the natural shape (no Form) cannot type-equal a PageData with Form, and adding `Form any` to the user's struct double-declares — there is no shape that satisfies both. `injectFormField` in `server/actions.go` already handles missing-Form via reflection, so the runtime work is done; only the codegen contract is wrong.
- PageData inference (`internal/codegen/pagedata.go`) reads the **first** `return` composite literal whose `Type` is a `*goast.StructType`. A type-aliased `result{}` return defeats inference because `lit.Type` becomes `*goast.Ident`. Inline anonymous struct literals at every return site are mandatory until inference grows.
- Codegen also cannot resolve user-defined named types (e.g. `[]PostSummary`) referenced in the Load return signature: the gen package imports `usersrc.routes` but the inferred field types are written verbatim into PageData, which lives in the gen package and lacks the alias. Anonymous struct fields work; named user types break the build with `undefined: PostSummary`.
- The malformed-import-path issue (`src/routes/[slug]`) means `go list ./...` from the playground only sees `cmd/app`. That is fine for the workspace toolchain (codegen reads source via `go/parser`), but unit tests cannot live under `src/routes/[slug]/` — the package is unreachable to the Go test runner.
- Multi-agent worktrees share the parent filesystem. Editing absolute paths under the parent worktree (`/Users/user/Workspaces/binsar/svelte-adapter-golang/...`) writes to whichever branch the parent currently has checked out, NOT the agent's isolated worktree. All work must happen under `.claude/worktrees/agent-<id>/...` to land in the right branch.

### Self-rules

1. **PageData inference assumes inline anonymous struct returns at every return site.** Type-aliased struct returns (`type result = struct{...}; return result{}, nil`) silently produce empty PageData. Never use a type alias as the Load return; spell the struct literal at every return statement until pagedata inference is upgraded.
2. **Field types in the inferred PageData must be self-contained.** Anonymous struct slice elements (`[]struct{...}`) cross the gen-package boundary; named user types (`[]PostSummary`) do not because the gen package never aliases them. Until codegen learns to forward user types, write inline anonymous structs all the way down.
3. **Don't combine `Load() returning struct{...}` with `var Actions = kit.ActionMap{}` until #143 lands.** The codegen-emitted PageData adds `Form any` and the render adapter strict-casts; user code has no satisfying shape. Smoke routes under Actions will return 500 with `PageData type mismatch`. Document the gap; do not paper over it.
4. **A playground that surfaces a framework gap should ship the gap-triggering source AND a follow-up issue with reproducer + acceptance criteria.** The next contributor should be able to fix the framework, point at the playground, and watch the smoke turn green. Hiding the gap costs more than documenting it.
5. **`go run github.com/binsarjr/sveltego/cmd/sveltego compile` must run before `go list ./...` or `go test ./...` in any playground.** The mirror tree under `.gen/usersrc/` is what the workspace toolchain sees; without it the whole module cannot resolve.
6. **Pick disjoint default ports across playgrounds.** Basic on `:3000`, blog on `:8080`. Future playgrounds: `:8081`, `:8082`, ... so a developer can run all of them in parallel without a port-clash dance.
7. **Always edit and commit from inside the agent's isolated worktree (`.claude/worktrees/agent-<id>/...`).** Parent-path absolute paths point at whatever branch the parent has checked out, which is not the agent's branch. Mixing paths leaks files into `main` and stomps other agents' staged work.
- **JS signed-int arithmetic and Go uint32 modular arithmetic produce identical bit patterns for this hash.** `((h << 5) - h)` overflows to negative in JS, XOR with `charCodeAt` runs on 32-bit ints, the final `>>> 0` reinterprets as unsigned. Running the entire fold in `uint32` reproduces every intermediate bit because addition/subtraction/shift are modular under both signed and unsigned representations — only the sign bit's interpretation changes, and the XOR cleans that. Bonus: avoids `int32 -> uint32` conversions that gosec flags as G115. Lesson: when a port involves signed↔unsigned conversion at the boundary, run the whole fold in unsigned; the bits agree.
- **CSS scope class application is an AST mutation, not an emit-time concern, when element.go is FORBIDDEN.** The natural place to intercept "every element should get class=svelte-xxxxx" is `emitOpenTag` in element.go — but that file is owned by other parallel phases. Mutating Element.Attributes in a pre-emit pass (`applyScopeClass(frag, hash)`) keeps the entire feature in one new file (style.go) and uses the existing emit pipeline unchanged. The mutation is idempotent (reuses static class slot if present, wraps dynamic into interpolated). Cleaner than threading a `scopeClass string` through Builder and reading it from element.go.
- **Removing nodes from the fragment is the right way to "consume" them in extraction passes.** `<svelte:options>` and `<style>` both need to disappear from the fragment so `emitChildren` does not try to render them. Filtering in-place (`kept := frag.Children[:0]`) followed by clearing the trailing slots avoids creating a parallel "skip these" set. Same pattern used by other languages' AST passes. Pairs with the codegen invariant that a node either lowers to source or gets extracted before lowering — never both.
- **`<svelte:options>` is parsed as a regular `*ast.Element` with `Name="svelte:options"`. The "specialness" lives in codegen, not the AST.** First considered adding an `*ast.SvelteOptions` node so the parser would surface it as a typed top-level. Two reasons against: (1) the parser already emits Element for `<svelte:*>` namespace prefixes and downstream codegen handles each by name match, and (2) parser is FORBIDDEN this phase. Treating it as Element + name match keeps codegen the validation authority and matches how `<svelte:body|window|document|head|component>` are already handled. AST node types are reserved for shapes that need different tree topology, not for special semantics.
- **Boolean `<svelte:options>` attrs need pointer fields to distinguish "absent" from "explicitly false".** A `bool` field with default `false` cannot tell `<svelte:options runes={false}/>` (opt-out, must validate) apart from `<svelte:options/>` (no opinion, runes are project default). Pointer-to-bool with three states (nil/&true/&false) is the minimum. Using tristate enums would be heavier; `*bool` is conventional Go for this pattern.
- **Acceptance items deferred when their implementation site is FORBIDDEN.** `namespace="svg"` requires element.go changes (skip void-element rules, emit `<path/>`-style self-closing). element.go is FORBIDDEN this phase, so the option is captured + plumbed but does not yet alter emission. Filed as follow-up. Same for selector-target matching in #54 — captured and parameterized as MVP "scope-all-elements" simplification; follow-up issue owns the selector matching algorithm. Phased delivery beats touching FORBIDDEN files.

### Self-rules

1. **When porting a hash function from JS to Go, run the entire fold in `uint32`, not `int32`.** JS signed/unsigned dance is finalized by `>>> 0`; uint32 mod-2^32 produces the same bit pattern at every step and avoids gosec G115 conversion warnings. Verify with Node-driven golden vectors hardcoded into the test — never call out to Node from CI.
2. **Pin the upstream version in a doc comment at the function site.** "Snapshot pinned 2026-04-30, file `packages/svelte/src/utils.js`." When upgrading, the next dev re-checks the function. No external pin file needed.
3. **When the implementing site is FORBIDDEN, feature surface = AST mutation pass before emit.** Scoped-class application mutates Element.Attributes in a pre-emit walk. Same trick works for any "tag every element with X" requirement. Element-level emit code is unchanged; the new behavior lives in the new file.
4. **Boolean compile-time options that can be explicitly opt-in OR opt-out get pointer fields.** `*bool` distinguishes nil/false/true. Avoids the three-way confusion of "not set" vs "set to false".
5. **`<svelte:*>` elements live as `*ast.Element` with name match in codegen — do not invent typed AST nodes for each.** Element + name match is the existing convention (`svelte:body`, `svelte:window`, `svelte:component`). New typed nodes are reserved for cases where tree topology differs.
6. **In-place fragment filtering is the right "consume this node" pattern.** `kept := children[:0]` + clear-trailing pointers + `frag.Children = kept`. No parallel "extracted" set, no skip-flag on the iterator.
7. **Defer acceptance items whose implementation requires touching a FORBIDDEN file. File a follow-up issue and document the deferral in the PR body.** Better to ship the MVP with explicit gaps than to either (a) violate the file boundary and break parallel agents or (b) hold the entire PR until the FORBIDDEN file frees up.
