## Phase 0y — CSP nonce + sitemap/robots helpers (#66 + #91) (2026-04-30)

### Insight

- **The CSP middleware fits cleanly above Handle, not inside it.** Header writes performed before `runHandle(ev, resolve)` survive every downstream path: success `writeResponse`, error `respondWithError`, short-circuit `errServerRouteWrote`, and 404 `writeSafeError`. A user-installed Handle hook still sees the nonce on `ev.Locals` because `applyCSP` runs first; their hook can override the header by Set-ing it on the response. One source of truth, no per-path duplication.
- **Go's HTTP transport silently trims leading whitespace from header values.** The `kit.NonceAttr(ev)` helper returns ` nonce="..."` (leading space deliberate) so users embed it directly into a `<script` tag mid-attribute. The first integration test asserted the leading space round-tripped through `resp.Header.Get`; it doesn't. The wire-trimmed view is normal — the helper's contract is the in-memory string, not what survives a transport. Test the helper's return value directly OR strip the leading space before sending; do not couple the helper's documented shape to net/http's normalization rules.
- **A 16-byte / 22-character base64 nonce is the OWASP sweet spot.** `crypto/rand.Read` into a `[16]byte` plus `base64.RawURLEncoding` skips padding and produces a token safe for HTML attributes without escaping. RawStdEncoding would also work but introduces `+` and `/` which look unfriendly mid-attribute even though they're attribute-safe. Pick one encoding (RawURL) and stay consistent across the helper, the integration test, and the docs.
- **Default directives are rebuilt per request, not memoized.** `DefaultCSPDirectives()` returns a fresh map each call so `BuildCSPHeader` can mutate `script-src` to splice the nonce token without leaking state across requests. Memoizing the defaults at package scope and then `append`-ing the nonce into the cached slice is a footgun: `append` may reuse the cached backing array on the second call and produce a header containing two nonces. The "always copy on read" rule is cheaper than tracking when copy-on-write is needed.
- **`bytes.Buffer.WriteRune` is the right primitive for XML escaping.** SitemapBuilder iterates the input string with `for _, r := range s` so the loop sees runes, not bytes. Calling `buf.WriteByte(byte(r))` would silently corrupt non-ASCII codepoints; `buf.WriteRune` writes the full UTF-8 sequence. Cheap reminder: when a string-iteration loop's loop variable is `r` (rune), the writer call must match the type.
- **Hydration nonce wiring is a future-phase concern, not a same-PR change.** The hydration `<script id="__sveltego__" ...>` tag does not exist in the runtime yet — it lands with #34 (Vite client). Trying to add nonce attribution speculatively means inventing an emit point that doesn't exist, then rewriting it once #34 chooses a different shape. The right move: ship `kit.NonceAttr(ev)` for user-emitted scripts now, add the hydration tag with `nonce="..."` baked in when #34 lands, and document the deferral explicitly so reviewers don't expect both halves in this PR.
- **Builder API for robots.txt mirrors the file format, not a graph data structure.** `r.UserAgent("X").Allow("/").Disallow("/admin/")` reads top-down like the on-disk file. Encoding the same intent through a `Group` struct with `(*Group).Allow(...)` would force the user to remember which group they're holding; the chainable form keeps the current group implicit and switches it on each `UserAgent` call. The implicit `*` group when a rule is added before any UserAgent matches the spec's "rules without a User-agent attach to all crawlers" reading and saves the most common single-group case from a redundant `.UserAgent("*")`.

### Self-rules

1. **Middleware that needs to set headers on every response path runs before the path branches.** Add the side effect in `handle()` immediately after `NewRequestEvent`, never inside one of the path-specific writers. Verifying coverage means writing one integration test per terminal path (success / error / 404 / hook short-circuit) and asserting the header is present.
2. **When the user-facing helper returns a string with whitespace prefixes/suffixes, document and test the contract directly — not through net/http round-trip.** Leading-space attributes are a normal HTML idiom; net/http's header canonicalization is not the audience. Unit-test the helper's return value; integration-test the *behavior* the user gets when they paste the string into HTML output.
3. **Maps returned from "default config" functions return fresh allocations every call.** Don't cache and don't share a returned map between requests. The cost of one heap allocation per request is negligible against the cost of debugging a header that contains two nonces because `append` reused a backing array.
4. **Defer wire-up to a not-yet-existing emit point with an explicit note in the PR description.** When the helper lands but the consumer is still waiting on a future issue, both the PR and the lessons entry call out the deferral and identify the issue that will close the loop. Never stub a placeholder emit "for symmetry" when the real shape is unknown — it'll be wrong shape when the consumer lands and the stub becomes a rebase hazard.
5. **Crypto choices in framework-owned helpers go through the standard library, not third-party utilities.** `crypto/rand` + `encoding/base64` covers nonce generation in 5 lines with zero dependency surface. The instinct to reach for a "random string" helper from an internal utils package is wrong — `math/rand` finds its way into those packages eventually, and CSP nonces with predictable RNG are an XSS bypass not a security feature.
