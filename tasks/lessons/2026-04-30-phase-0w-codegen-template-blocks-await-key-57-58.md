## Phase 0w — codegen template blocks ({#await} + {#key}) (#57 + #58) (2026-04-30)

### Insight

- **AST + parser + dispatcher were already wired with TODO stubs.** Both `AwaitBlock` and `KeyBlock` had AST nodes, parser entry points (`parseAwaitBlock` / `parseKeyBlock`), and `emitNode` switch arms returning `// TODO`. The Phase 0w work was the lowering body plus position/diagnostic rules — same shape as Phase 0v. Always grep for the AST type before assuming greenfield: a prior phase may have left a stub waiting for the lowering to land.
- **`{#await x then v}` shorthand is a Svelte syntax our parser does NOT support.** Parser only accepts the longhand `{#await fn()}{:then v}...{:catch e}...{/await}`. The first fixture pass used the shorthand from the issue body verbatim, parsed cleanly into an AwaitBlock with `Expr="fetchUsers() then users"` and empty Then/ThenVar — codegen emitted nothing surprising but the goldens looked wrong. Fix: rewrite fixtures to longhand. Lesson: when copying source examples from an issue, run the existing parser test corpus to confirm syntax support before assuming the parser implements every Svelte shape.
- **Pending branches with whitespace text must NOT trigger the no-streaming diagnostic.** Source like `{#await fn()}\n  {:then v}...` parses with a single whitespace Text node inside Pending. A naive `len(Pending) > 0` check rejects every well-formatted template. The right test is "any non-whitespace child" — implemented as `hasNonEmptyBody` via a small `isWhitespace` helper. The diagnostic still fires on a genuine pending branch with content.
- **`{#await}` Go lowering needs three Go-correctness fences.** (1) When then-branch is empty, force `thenVar = "_"` so we don't introduce an unused variable. (2) When catch-branch is empty, emit `return _err` (the surrounding Render method returns `error`, so propagation is built in). (3) When catch-branch carries a CatchVar, emit `<var> := _err` to consume `_err` and bind the user-chosen identifier. Skipping any of the three produces "declared and not used" errors from go/format. The wrapping `{ ... }` block scopes `_err` so nested awaits don't collide.
- **`{#key}` index is per-template counter, not per-render counter.** SSR emits `<!--sgkey:0:expr-->` markers with a static index assigned at codegen time, stored on the `Builder`. Each iteration of an enclosing `{#each}` reuses the same index (intentional — the boundary identity is template-static, multiple instances are normal). Hydration metadata (#35) maps index → expression text, so client diffing locates the boundary by index regardless of where it appears in the rendered HTML.
- **Phase 0t (#126) landed in main mid-work.** Rebase against origin/main was clean because file lists were genuinely disjoint at the function level: 0t added `GenerateErrorPage` + `ErrorPageOptions` types and an `errorpage` skip in TestFixtures; 0w added `emitAwaitBlock`/`emitKeyBlock` and a `keyCounter` field on `Builder`. The shared `emitNode` switch picked up both phases' arms with no semantic conflict. Concurrency note from the brief held: "rebase is cheap because file lists are disjoint" — confirmed.

### Self-rules

1. **Before authoring fixtures from an issue body, parse a candidate against the existing test corpus.** The issue body is a spec, not a syntax oracle. Our parser may implement a subset of Svelte's surface; longhand vs shorthand mismatch silently produces wrong goldens. One quick `go test -run TestAwait` confirms the shape before fixture generation.
2. **Diagnostics that need to fire on "any meaningful content" use a whitespace-aware predicate, not `len(...) > 0`.** Source formatting produces stray whitespace Text nodes between block boundaries. `hasNonEmptyBody` walks children once and skips pure-whitespace text. Reuse this when adding future "branch unsupported under <feature>" diagnostics.
3. **Generated Go that introduces a fresh local must consume that local in every reachable path or alias it to `_`.** For `{#await}`, the then-branch may be empty (catch-only template); in that path, the result name has no consumer and Go's compiler rejects it. Switch the then-binding to `_` upfront when len(Then)==0. Same rule applies to any future "split a single value across branches" lowering.
4. **Per-template counters live on `Builder`, not as package-level state.** `keyCounter int` is a struct field so each Generate / GenerateLayout / GenerateErrorPage call starts at 0 and counters don't bleed across templates. Resist the urge to make it a global — concurrent codegen runs would corrupt indices and tests would flake.
5. **When concurrency note says "file lists are disjoint", verify by `git diff --stat` after rebase rather than assuming.** Phase 0t's commit listed both `codegen.go` and `codegen_test.go` (files I also edited), but the line ranges didn't overlap. A 30-second post-rebase build + test run confirms no semantic conflict; skip that step and a silent merge of two `case *ast.AwaitBlock:` arms goes undetected by git but blows up at runtime.
