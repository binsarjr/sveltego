## Phase 0r — page options + REST endpoint dispatch (#79 + #29) (2026-04-30)

### Insight

- **TrailingSlash policy lives on the matched route, not on the URL — but the URL drives match selection.** The chicken-and-egg flips when the policy flips: a route declared `/about` with `TrailingSlashNever` still receives requests for `/about/` (the radix tree returns a miss), and a route declared `/about` with `TrailingSlashAlways` cannot redirect away from itself without first matching. Resolution: try the match as-is; on miss, retry with the toggled trailing slash; if the retry hits, `trailingSlashRedirect` then fires the canonical 308 against the now-known route policy. The retry only runs on miss, so canonical traffic still pays a single tree lookup.
- **`Options` defaults to its zero value on legacy routes.** The router's `Route` struct carries `Options kit.PageOptions`; pre-Phase-0r manifests never populate it. The zero value sets `SSR=false` and `TrailingSlash=Default`, which would break older routes if SSR=false were treated as "skip render". The pipeline distinguishes "SSR explicitly false" from "no options resolved" by comparing the entire struct against `kit.PageOptions{}`. Cleaner than a sentinel pointer; survives `if opts == (kit.PageOptions{})` cleanly because the struct is comparable.
- **Cascade resolution belongs in codegen, not in the runtime.** SvelteKit-style page options cascade outer→inner across the layout chain. Computing the effective value at request time would force the pipeline to walk the layout chain on every match. Resolving at codegen and storing the materialized `kit.PageOptions{...}` literal in each manifest entry keeps the request path branch-free: one struct read, one comparison, decide the route.
- **Const scanner mirrors hookscan but recognizes `*ast.GenDecl` with `Tok=token.CONST`.** `go/parser` exposes top-level constants as `*ast.GenDecl` with `Tok=token.CONST`; each `*ast.ValueSpec` lists Names and Values in lockstep. Bool literals come back as `*ast.Ident{Name:"true"|"false"}`; selector-form `kit.TrailingSlashAlways` comes back as `*ast.SelectorExpr`. Bare `TrailingSlashAlways` (dot-import case) appears as `*ast.Ident`; recognizing both forms keeps the scanner honest without a full type-check.
- **Renaming `emitServerStub` to `emitRESTRoute` was the right move, not adding a parallel function.** The placeholder existed only to satisfy `route.HasServer` in earlier phases. Keeping it alongside a new dispatcher emitter would double-write the same `server.gen.go` file. Step 0 (delete dead code) cleared it before the new emitter landed; the manifest's `Server: <pkg>.Handlers` reference now resolves to the dispatcher map populated by the user's verb functions, not a TODO comment.
- **Issue body referenced `+server.go`, but ADR 0003 amendment landed `server.go` (no plus).** The brief flagged this convention conflict explicitly: the file convention was settled in Phase 0i-fix and `routescan/scan.go` already gates on `server.go`. Built against the actual scanner contract; PR description carries the convention note so anyone re-reading #29 sees the file naming corrected at codegen time.
- **`<div id="app"></div>` is the SSR=false body, not a synthesized full page.** Per #79 acceptance: "renders empty shell server-side". The framework cannot speculate on layout chrome for SSR=false routes (layouts may declare data that never gets loaded). Emitting just the mount point keeps the contract small and lets the Vite client (#34) take over on first hydration.

### Self-rules

1. **When request shape (path) and resource shape (route) disagree, retry the match before declaring 404.** A miss with a togglable suffix (trailing slash, case folding, etc.) deserves one retry under a canonicalization rule recorded on the matched resource. The retry is bounded; an unconditional second pass is fine when the first miss is cheap. Never paper over the disagreement by registering both forms in the radix tree — duplication breaks `Pattern`-keyed lookups elsewhere.
2. **Comparable struct zero values double as a "not set" sentinel for runtime feature gates.** When a value type is comparable and its zero state is meaningful in the legacy code path, `if v == (T{})` is cheaper than threading a `*T`. Document the convention at the comparison site so future readers understand "zero means legacy, not 'all defaults explicit'".
3. **Cascade resolution happens at codegen, not at request time.** Any feature that walks the layout chain (page options, layout-level cookies-namespace, layout-level prerender hints) gets resolved during `Build` and materialized into the manifest entry. The runtime reads scalars; never re-walks.
4. **AST const-decl scanning recognizes both `*ast.SelectorExpr` (qualified) and `*ast.Ident` (dot-import) forms for typed constants.** Only typed boolean and enum constants need this; integer literals or string constants are simpler. Reject unrecognized identifiers explicitly so a typo (`TrailingSlashWeird`) fails the build instead of degrading to the zero value.
5. **When renaming an old placeholder emitter to a real one, delete the placeholder first.** Step 0 of the project's structural-refactor rule. The dead function sits in the diff for one commit alongside the manifest.go updates that made it obsolete; a separate cleanup commit is overkill for a single function. Deleting in the same commit as the replacement keeps reviewers' eyes on the new emitter rather than the parallel "emitServerStub still exists" question.
6. **File convention disagreements between issue body and ADR amendment go to the ADR.** Issue #29 wrote `+server.go`; ADR 0003 (Phase 0i-fix) wrote `server.go`. The PR description carries the discrepancy note so anyone reading #29 alongside the PR sees that the convention was already settled. Never edit a closed RFC issue body to retroactively match later amendments — keep the change log in the ADR.
