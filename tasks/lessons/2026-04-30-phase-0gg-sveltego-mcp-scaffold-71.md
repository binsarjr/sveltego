## Phase 0gg — sveltego-mcp scaffold (#71) (2026-04-30)

### Insight

- **A handcrafted JSON-RPC 2.0 layer is the right scaffold when the SDK landscape is in flux.** MCP's official Go SDK is young and the wire format is a tight subset of JSON-RPC: `initialize`, `tools/list`, `tools/call`, `resources/list`, `resources/read`, plus `notifications/initialized`. About 200 LOC of `bufio.Scanner` + `json.Encoder` covers the protocol surface this server needs and removes a third-party dependency that would otherwise need version pinning + upgrade discipline. Trade is a small handwritten protocol layer for a shrinking-import-graph and an SDK we control. Document the choice and revisit when the official SDK stabilises.
- **Go's `internal/` rule is a hard wall, not a guideline.** The MCP module lives under `packages/mcp/` and `validate_template` would naturally call `packages/sveltego/internal/parser.Parse`. Go's compiler rejects this at build time — `internal/` is only importable from siblings of the parent. Re-discovered the rule the hard way (toy import test in `/tmp` returned "use of internal package not allowed"). Fix is one of: thin re-export under `packages/sveltego/exports/parser`, shell out to the `sveltego` CLI's `compile` subcommand, or move the parser out of `internal/` if it's stable. Stub the tool with a clear follow-up message so MCP clients see the gap rather than silent success.
- **`ast.Package` + `go/doc.New` is the right primitive for symbol introspection even though `ast.Package` is deprecated.** `go/doc.New` accepts only `*ast.Package`; the deprecation tells you to use `go/types` for type-checking but `go/doc` is still the canonical extractor for godoc + decl text. Suppress with a single targeted `//nolint:staticcheck` and a one-line rationale comment, not a global lint exclusion. The output (`func Redirect(code int, location string) error` + the leading godoc paragraph) is exactly what an LLM client wants in a single text block.
- **Snippets (`get_example`) need an explicit byte cap and a `node_modules`/`.gen` skip walk.** A naive `filepath.WalkDir` over `playgrounds/basic/` would balloon a tool response past any reasonable client buffer the moment dependencies land. 100KB cap + `fs.SkipDir` for known-large trees keeps the shape predictable and the response readable. The cap message ("…(truncated at 100KB)") is part of the tool's contract, not an afterthought.
- **MCP stdio framing is line-delimited JSON, not LSP-style Content-Length.** Easy to confuse because both are JSON-RPC over stdio in editor tooling. MCP picked simpler framing; one message per line; a `bufio.Scanner` with a 4MB `Buffer()` ceiling handles every realistic tool argument including pasted multi-KB Svelte snippets. Don't reach for `net/textproto` headers.

### Self-rules

1. **Before importing a path that contains `/internal/`, verify the consuming module is a descendant of the parent of that `internal/`.** Go enforces this at compile, not import-resolution, so a stale belief survives reading the source. Run a one-file `go build` smoke against the proposed import before designing the call site. If the boundary blocks the call, design a re-export at the package boundary (`packages/<pkg>/exports/<sub>`) or a CLI shell-out, and stub the consumer with a follow-up.
2. **For protocol-style scaffolds where the SDK is young, write the JSON-RPC layer by hand and document the choice.** A 200-LOC handwritten dispatch is auditable, version-stable, and avoids API churn from a third-party. Revisit when the SDK ships v1; until then, the trade is favourable.
3. **Tool responses that walk a directory tree get a hard byte cap and a known-skip list.** `node_modules`, `.gen`, `vendor`, `.git`. Use `fs.SkipDir` and a running `b.Len() >= maxBytes` check. The truncation marker is part of the response, not a client problem.
4. **Suppress targeted deprecations with a one-line `//nolint:linter` and a rationale comment.** Global excludes hide real issues. `ast.Package` is deprecated for type-checking but remains the only input to `go/doc.New`; that's a one-call exception, not a project-wide allowance.
5. **Validate slug and example-name inputs against the configured base directory by recomputing `filepath.Join(base, clean)` and asserting the result still has `base` as prefix.** A string-only `..` filter is necessary but not sufficient on platforms with mixed separators or symlinks. Trust the OS join + prefix check.
