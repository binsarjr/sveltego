> Note: dated 2026-05-01 to keep the journey adjacent to the
> 2026-05-02 decision lesson in the index. Decision codified

### Insight

- A 9-phase rollout under a single tracking issue ([#422](https://github.com/binsarjr/sveltego/issues/422)) made it possible to ship Option B end-to-end in a single working session because each phase's deliverable was a hard interface for the next: the JSON envelope (Phase 2) gates the emitter (Phase 3), the helpers package (Phase 4) gates pattern entries, the lowerer (Phase 5) gates pipeline integration (Phase 6), and the fallback escape hatch (Phase 8) replaces Phase 6's transitional soft-fallback. Sequencing wasn't optional — it was the contract between agents.
- The "no JS runtime on the server" invariant ([ADR 0005](../decisions/0005-non-goals.md)) survived intact. Node ran at build time for Acorn JSON-AST emit (Phase 2) and at request time only for routes that opt in via the `<!-- sveltego:ssr-fallback -->` HTML comment in `_page.svelte` (Phase 8). The deployed Go binary plus `static/` is still the entire deployable.
- **Vendored Acorn was the load-bearing call.** Hand-rolling a JS parser in Go was the largest single line item in the original Option B LOC estimate (~1,500 LOC, build-time-only). Vendoring Acorn — Svelte's own parser dependency — in the existing build-time sidecar dropped Option B's build-time LOC from ~4,000 to ~2,500 and eliminated a class of "we got the JS grammar wrong" bugs at the source.
- **Hard-error-at-build beat silent fallback.** Phase 6 had to ship a transitional soft-fallback because Phase 8 hadn't landed yet; the warning log named the missing annotation. As soon as Phase 8 landed, soft-fallback was deleted and hard-error became the default. Coverage maps stay honest only when degradation is opt-in.
- **Spec-vs-reality deltas are normal and should be captured at the source.** Phase 3's emitter discovered that Svelte 5.55.5's compiled output differs from the spec in three load-bearing ways: `$$renderer.push(template_literal)` is the dominant emit (not `$$payload.out += '...'`), `$.escape` not `$.escape_html`, hand-rolled C-style `for` not `$.each(...)`. Phase 9 wrote those deltas back into [`tasks/specs/ssr-json-ast.md`](../specs/ssr-json-ast.md) so future maintainers don't re-discover the same surprises. The spec is now reality-checked.

### Self-rules

1. When a multi-phase track's phases share an interface contract (JSON envelope, scope enum, lowerer signature), document the contract explicitly in `tasks/specs/` BEFORE Phase 1 closes. Each downstream phase's quirk file references the spec; the spec is the single point of truth.
2. When a transitional soft-fallback is introduced for a phase ordering reason, file a CRYSTAL-CLEAR issue for its deletion and link it from the soft-fallback site itself (warning log, code comment). The transition window must be visibly bounded.
3. After landing a generator, **reality-check the generator's input spec** in the docs sync phase. If the real input differs from the spec assumed at design time, update the spec — don't update the generator to match a wrong spec, and don't carry an incorrect spec forward as documentation.
4. When extending an existing build-time tool (the SSG sidecar) to take on adjacent responsibility (Acorn JSON AST emit + long-running fallback mode), structure it as **modes** (`--mode=ssg`, `--mode=ssr`, `--mode=ssr-serve`) not separate binaries. Single npm install, single pinned Svelte minor, single dependency surface.
5. Per-phase quirk files (`/tmp/phaseN-quirks-for-...md`) handed off between agents are a powerful coordination tool when the phases are sequential and bounded. They are NOT a substitute for code-level documentation — they capture *intent* that won't make it into the diff (why a soft-fallback exists, why an unknown shape is intentional, what's deferred).
6. When promoting a runtime package out of Experimental, gate the promotion on either v1.0 RC or a named successor stability story; never on "it's been a while". `runtime/svelte/server` and `runtime/svelte/fallback` both stayed Experimental at the end of Phase 8 / 9 because lowerer carryovers ([#440](https://github.com/binsarjr/sveltego/issues/440), [#443](https://github.com/binsarjr/sveltego/issues/443)) may still extend the helper call patterns.

### Phase-by-phase headlines (chronological)

- **Phase 1 — ADR 0009 codified.** Three sub-decisions captured at decision time: vendored Acorn, hard-error-at-build, lock-one-Svelte-minor. Decision lesson: [2026-05-02-ssr-option-b-decision.md](2026-05-02-ssr-option-b-decision.md). Closed [#423](https://github.com/binsarjr/sveltego/issues/423).
- **Phase 2 — Node sidecar acorn JSON-AST extension.** Vendored Acorn 8.16.0 and Svelte 5.55.5 in the existing SSG sidecar; added `--mode=ssr`. Determinism guard rails: sorted keys, no timestamps, no absolute paths, never `dev: true`. Spec at [`tasks/specs/ssr-json-ast.md`](../specs/ssr-json-ast.md). Closed [#424](https://github.com/binsarjr/sveltego/issues/424).
- **Phase 3 — Go pattern-match emitter scaffold + 30 priority shapes.** New package `internal/codegen/svelte_js2go/`. `Options.Rewriter` interface as the Phase 5 hook. `Scope` + `LocalKind` exported enum. `TestTranspile_Determinism` enforces byte-identical output. Surfaced reality deltas vs spec (renderer.push vs payload.out, `$.escape` vs `$.escape_html`, hand-rolled for vs `$.each`). Closed [#425](https://github.com/binsarjr/sveltego/issues/425).
- **Phase 4 — runtime/svelte/server helpers.** ~30 pure functions mirroring `svelte/internal/server`. EscapeHTML / EscapeHTMLAttr split (CONTENT_REGEX vs ATTR_REGEX, no `>` or `'` either path) is a load-bearing intentional mirror of upstream. Closed [#426](https://github.com/binsarjr/sveltego/issues/426).
- **Phase 5 — property-access lowering.** `data.name` → `data.Name` via the typegen JSON-tag table. Strict mode hard-errors on unsupported expression shapes. Optional-chain lowering uses inline IIFE guards (Phase 4 surface kept closed). Closed [#427](https://github.com/binsarjr/sveltego/issues/427).
- **Phase 6 — pipeline integration.** SSR live end-to-end in `playgrounds/basic`. Typed-struct receipt swap (`PageData` instead of `map[string]any`). Build-context propagation through `codegen.Build(ctx, opts)`. Transitional soft-fallback shipped because Phase 8 hadn't landed; warning log named the missing annotation. Closed [#428](https://github.com/binsarjr/sveltego/issues/428).
- **Phase 7 — golden corpus + ssr-stress playground.** Expanded the corpus from 30 to 80+ shapes. Added `playgrounds/ssr-stress/`. Cross-check fixtures (`runtime/svelte/server/testdata/cross/`) diff Go output against captured Node output. Bench harness extended with SSR scenarios. Surfaced [#443](https://github.com/binsarjr/sveltego/issues/443) (snippet hoisting + `{@const}` non-bool). Closed [#429](https://github.com/binsarjr/sveltego/issues/429).
- **Phase 8 — fallback escape hatch.** `<!-- sveltego:ssr-fallback -->` annotation routed through long-running Node sidecar (`--mode=ssr-serve`). HTTP on 127.0.0.1 ephemeral port; ready prefix `SVELTEGO_SSR_FALLBACK_LISTEN port=NNN` on stderr. In-memory LRU+TTL cache. Supervisor restarts up to 3× with exponential backoff. Soft-fallback from Phase 6 deleted; hard-error became default. Closed [#430](https://github.com/binsarjr/sveltego/issues/430).
- **Phase 9 — post-implementation doc sync (this PR).** README + CLAUDE.md + AGENTS.md + spec + per-package STABILITY/CLAUDE.md + playground READMEs + lessons file. Reality-checked the spec doc with Phase 3's deltas. Milestone-count cross-update. Closed [#431](https://github.com/binsarjr/sveltego/issues/431).

### Named surprises during execution

- **Worktree hijack.** Phase 9 inherited a worktree that had been used for an earlier phase's branch (`phase/ssr-fallback-430`). Recovery was to `git fetch origin main` then `git checkout -b phase/ssr-docs-431 origin/main` from inside the worktree (`git checkout main` directly fails because main is checked out in the parent worktree). Worth memorising for any agent that picks up a multi-phase worktree.
- **Scope-cut acknowledgement around layout chain.** Phase 6's pipeline integration deferred the children-callback ABI for layouts because it was a genuinely independent issue, not a cross-phase contract. Filed as [#440](https://github.com/binsarjr/sveltego/issues/440) and called out in every phase's quirk file thereafter. Phase 7 added the test fixture (`playgrounds/ssr-stress/layoutdeep/...`) so the deferred work has a target; #440 stays open for the implementation.
- **Soft-fallback transition.** Phase 6 had to ship a soft-fallback transitionally because Phase 8 hadn't landed; the warning log named the missing annotation. As soon as Phase 8 merged, the soft-fallback was deleted and hard-error became the default. The transition window was clearly bounded by issue links and code comments, but it's worth flagging that "transitional soft-fallback" should always come with a deletion issue.
- **Side fix in `ssr_serve.mjs`.** `ERR_HTTP_HEADERS_SENT` race on the render-error path: JSON-encode payload BEFORE `writeHead`; gate the catch-block `writeHead` on `!res.headersSent`. Phase 8 fixed this; the order is now load-bearing.
