## Phase 0q — hooks group (#26 + #27 + #80 + #81) (2026-04-30)

### Insight

- **No generated `main.go` in this architecture.** The brief assumed a generated `main.go` would call `hooks.Init(ctx)`, mirroring SvelteKit's adapter output. Reading `playgrounds/basic/cmd/app/main.go` first showed that user code authors `main()` and wires `server.New(server.Config{...})` itself; codegen produces only `.gen/`. Right answer: expose `gen.Hooks() kit.Hooks` from codegen, accept `Config.Hooks`, and run `Init` inside `Server.ListenAndServe` (and an explicit `Server.Init(ctx)` for callers managing startup themselves). One factory call from user main.go threads everything; nothing is generated outside `.gen/`.
- **Handle wrapping the entire pipeline forces a buffered Response model for page routes.** The pre-0q pipeline wrote directly to `http.ResponseWriter` from the render buffer — no intermediate Response value. SvelteKit's Handle returns a Response, post-modifying headers and (theoretically) the body. To preserve that contract for page routes, `renderPage` now returns a `*kit.Response` carrying status/headers/body; the pipeline writes once at the very end. +server.go handlers cannot fit this model (they use raw `http.HandlerFunc` and stream through w directly), so resolve returns `errServerRouteWrote` for them and the writer step is skipped — Handle on +server.go routes can short-circuit and inspect Locals/URL but cannot post-modify headers. Documented as MVP scope.
- **Sentinel branch order in handlePipelineError matters.** RedirectErr / HTTPErr / FailErr / SafeError must each be tried via `errors.As` *before* delegating to the user's HandleError. Otherwise a user error implementing both a sentinel-shape and the broader interface gets misrouted (e.g. RedirectErr also implements HTTPStatus → without ordering it would lose the Location header). Same lesson as Phase 0m-X2; reaffirmed for the multi-branch HandleError path.
- **Backward-compat for the legacy `httpStatuser` interface.** Pre-0q tests (`TestServeHTTP_loadError`) asserted that any error type exposing `HTTPStatus() int` drove the response status without a sentinel. The new HandleError path collapses every uncaught error into a 500 SafeError by default (identity hook). To keep that pre-existing contract green, when the SafeError comes back as the canonical 500/InternalServerError pair AND the original error implements httpStatuser, the writer promotes the status from the interface. Surgical: only the default-identity case takes the legacy promotion; user-authored HandleError that returns a deliberate 500 is left alone.
- **Panic-recovery boundary is the framework's responsibility, not the user's.** Acceptance #27 demands that "Panic in Load is caught, passed to HandleError, returns 500 with logged ID." The pipeline wraps the Handle call in `defer/recover`, lifting any `recover()` value into a wrapped error so the rest of the error-routing logic is unchanged. That's the one place the project's no-panic rule (CONTRIBUTING) makes an exception, explicitly per AGENTS.md §3 ("forbidden: panic outside recovered HTTP middleware boundaries").
- **AST-based signature validation by arity is enough for MVP.** Full type-identity verification at codegen would require resolving `*kit.RequestEvent`, `kit.ResolveFn`, `*kit.Response`, etc. through the user's import scope — duplicating what `go build` already does. Counting parameters and results catches the obvious "func Handle()" mistake while letting the generated adapter compile its way to the real type error if signatures drift. Two-tier check: arity at codegen (fail-fast on garbage), full type at `go build`.
- **`hookssrc` mirror is a sibling of `usersrc` and `layoutsrc`, not a child.** Three independent mirror trees under `.gen/` (`usersrc/`, `layoutsrc/`, `hookssrc/`) keep the file-name conflicts away (every mirror would otherwise compete for `.gen/usersrc/hooks/hooks_server.go` vs route-encoded paths). Keeps each mirror's package name uniform too: `hookssrc` is a fixed identifier the wire code imports by alias, with no per-route encoding.

### Self-rules

1. **Read the generated/user-authored split before assuming a generated main.go.** Codegen produces `.gen/`; user owns `cmd/<app>/main.go`. New runtime contracts surface via factory functions on the gen package (`gen.Routes()`, `gen.Hooks()`, …) and through `server.Config` fields, never via a generated `main.go`. Anything that needs to run before `ListenAndServe` (Init) goes inside the server type, not in a generated wrapper.
2. **When a hook can both succeed and short-circuit, model resolve as `(*Response, error)` and reserve a sentinel error for "handler already wrote".** `errServerRouteWrote` keeps the pipeline writer's job simple: response is non-nil → write it; response is nil and err is nil → resolve already wrote (no-op); err is non-nil and not the sentinel → route to HandleError. Three branches, no overlap.
3. **Sentinel `errors.As` checks always run before the user-supplied HandleError.** Sentinel types (RedirectErr, HTTPErr, FailErr, SafeError) carry framework-defined semantics that HandleError cannot improve on. HandleError owns the *unsorted-yet* error set; sentinels are pre-sorted.
4. **Backward-compat shims for default-only behavior live in the writer, not in hook discovery.** When the user's HandleError absent (identity default), legacy interface checks (`httpStatuser`) still apply because pre-hooks code authoring relied on them. The shim is conditional on the SafeError being the canonical default-500 pair — user-authored HandleError returning 500 deliberately is not overridden.
5. **Panic recovery at the framework boundary is the documented exception to the no-panic rule.** Recover once, around the Handle call, and convert the panic value to an error. Any deeper recovery (per-route, per-Load) is the user's responsibility — wrapping every layer doubles the cost and the diagnostic noise.
6. **Mirror trees for user `.go` files use one fixed package name per concern, not per-route encoding.** `hookssrc` is the package every gen file imports by alias. Concern-bounded mirror trees (`usersrc/`, `layoutsrc/`, `hookssrc/`) keep filename collisions impossible without forcing a path-encoding scheme on top.
