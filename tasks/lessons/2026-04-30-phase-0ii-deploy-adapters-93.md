## Phase 0ii — deploy adapters (#93) (2026-04-30)

### Insight

- **`pwd` at session start does NOT bind subsequent absolute paths to the worktree.** The agent worktree is at `.claude/worktrees/agent-<hash>/` while the parent repo sits at the project root. Verifying `pwd` once gives a false sense of safety: every later `Write`/`Edit` with an absolute path bypasses cwd entirely. The bug surfaced as "git status shows nothing to commit" inside the worktree while the parent repo accumulated 16 untracked files. Always anchor file operations under `<worktree-root>/` and audit with `git -C <worktree-root> status` before claiming the phase is staged. Parent-repo's CLAUDE.md flags "stray cd to parent worktree" but the silent-write failure mode is the converse: stray *paths*, not stray cd.
- **Phase 0ff lesson #1 is load-bearing for every "create new top-level dir" prompt.** The user's spec said `adapters/<target>/adapter.go` and `cmd/sveltego-adapter/`. Those paths do not exist; CI fan-out, `go.work`, and release-please are wired for `packages/adapter-{auto,server,docker,lambda,static,cloudflare}`. Following the spec literally would have spawned a parallel tree the project's tooling cannot see. Map the spec to the existing layout: `adapters/server/adapter.go` → `packages/adapter-server/adapter.go`; `cmd/sveltego-adapter/` → `packages/adapter-auto/cmd/sveltego-adapter/` (mirrors how `packages/lsp/cmd/sveltego-lsp/` lives next to its dispatcher in Phase 0ff). Resolve the divergence in the PR body so reviewers see the constraint.
- **A common `Adapter` interface across modules adds friction without buying anything.** The user spec proposed an `adapters/adapter.go` interface every target satisfies. Because each adapter is its own Go module (`packages/adapter-{name}/`) with its own go.mod + release line, sharing the interface forces a shared "types" module that becomes a dependency every adapter must require. The pragmatic shape: each adapter exposes `Build(ctx, BuildContext) error` + `Doc() string` + `Name` const (small POD inputs, free of cross-module type sharing). The dispatcher (`adapter-auto`) imports each one and switches on target name. No interface, no shared module, identical user-visible API.
- **`aws-lambda-go` does not need to be a dependency of any sveltego module.** The spec hinted at a separate `cmd/sveltego-adapter/go.mod` to keep AWS deps isolated. Closer reading: the adapter EMITS Go source that uses `github.com/aws/aws-lambda-go`; it does not call those packages itself. The user's project gets the dependency via the emitted `.gen/lambda/main.go` + their own `go get`. The adapter library can stay zero-AWS. Same pattern works for `aws-lambda-go-api-proxy` and any future `cloudflare-workers-go` runtime.
- **CLI flag defaults must respect the dispatched adapter's preferred default.** First version of `sveltego-adapter build` set `--out` to `"dist"` always; lambda's adapter wanted `<root>/.gen/lambda` when OutputDir was empty. The empty-string-vs-explicit dance lives in the CLI: `defaultOutDir(target, explicit, root)` returns `""` for lambda (defer to adapter) and `<root>/dist` for everyone else. A blanket default in the flag declaration breaks per-target ergonomics.
- **Tests writing to `--out=dist` (relative) leak into the package directory and break `./...`.** The lambda dispatch test forgot to pass `--out`, so the relative default resolved against the test binary's cwd (the package dir) and dropped a `dist/main.go` next to the source. The next `go test ./...` recursed into that orphan dir, tried to build code referencing `aws-lambda-go`, and failed with "no required module provides package". Fix: give the CLI's lambda branch a real default (root-relative `.gen/lambda`); always anchor `--out` and `--root` via `filepath.Abs` before passing to adapter Build. Reproducible-build hygiene: tests that exercise file emission must use `t.TempDir()` for every path that doesn't already have an explicit absolute target.
- **Single-scope commit subject collapses multi-package edits via a higher-level scope.** The pre-commit hook's regex (`(scope)?` matching `[a-z0-9/_-]+`) accepts any single scope; the project convention is the package name. Six adapter packages in one commit have no single package scope, but the user's directive `feat(adapters)` (plural) is a valid scope — readers know it means "the adapter family", and the regex accepts it. Don't try to fit six modules into one of their names; pick the umbrella term and document it in the PR body.

### Self-rules

1. **Anchor every Write/Edit under the agent worktree root, not the parent repo.** The worktree path is `.claude/worktrees/agent-<hash>/`; verify the prefix on each absolute path before calling the tool. After every batch of file ops, run `git -C <worktree> status` (NOT `cd <worktree> && git status` — rtk may switch shells silently) to confirm changes landed in the worktree's tracked tree. If `git status` is empty in the worktree, every "successful" write went to the parent repo.
2. **Map prompt-supplied paths onto existing layout when CI/`go.work`/release-please already wires the alternative.** Run `grep -n "adapter\|<feature>" .github/workflows/ go.work release-please-config.json` at the start of any phase whose spec names a new top-level directory. If the directory is already wired elsewhere, deliver in the wired location and note the divergence in the PR body. Phase 0ff codified the rule; Phase 0ii reaffirmed it.
3. **Across-module type sharing requires its own module; if it doesn't, prefer per-module duplication of small PODs.** A common interface across N modules forces the consumers into a coordination dance every release. For shapes that fit on a screen, copy them. The dispatcher pattern (one module imports all, switches on name) keeps user-visible API uniform without forcing cross-module types.
4. **Code that only emits source for the user's project does not depend on the modules referenced in the emission.** Lambda adapter's `main.go` mentions `github.com/aws/aws-lambda-go`; the adapter library never imports it. The user's `go get` after they consume the emitted file is the right place for the dependency. This rule lets adapter modules stay slim and lets users pick AWS / GCP / on-prem without pulling unused trees.
5. **CLI flag defaults route through a per-target resolver, not a global default.** Generic `--out=dist` looks tidy until one target wants `<root>/.gen/<name>`. Implement `defaultXxx(target, explicit, root)` that returns `""` (defer) or a target-appropriate value. Same pattern for any flag whose semantically-correct default depends on the dispatched target.
6. **File-emitting tests use `t.TempDir()` for every path the SUT might write to.** Relative defaults silently anchor against `os.Getwd()`, which during `go test` is the package dir. Orphan output trees survive across runs and contaminate `./...` recursion. Pass absolute paths into the SUT; cleanup is automatic via `t.TempDir`.
7. **Multi-module commits use an umbrella scope (`adapters`, `runes`, `hooks`) when each individual package name is too narrow.** The pre-commit regex accepts any `[a-z0-9/_-]+` scope. A plural / family-name scope reads as "this commit touches the named family"; readers do not have to scan the file list to understand. Document the scope choice in the PR body so the next author follows the same convention.
