## Phase 0nn — static asset precompression (#41) (2026-04-30)

### Insight

- **Don't compress at request time; serve precompressed siblings.** The issue spec calls for `.br`/`.gz` produced at build, the runtime's job is content negotiation only. That decouples the hot path from any compression library, keeps the runtime dependency-free (no `andybalholm/brotli` in `go.mod`), and lets Caddy/Nginx-style speed fall out naturally. The handler's responsibility is exactly: pick the variant, set the encoding header, delegate I/O to `http.ServeContent`.
- **`net/http`'s default transport silently rewrites compressed responses on the client side.** When a request goes out without a manually-set `Accept-Encoding`, the transport injects `Accept-Encoding: gzip`, and on the response it transparently decompresses + strips `Content-Encoding`. This guts gzip-vs-raw test assertions — the body becomes empty (decompression of fake bytes fails) and the encoding header disappears. Tests that exercise compression *must* use a transport with `DisableCompression: true`.
- **`path.Clean` collapses `..` silently — it cannot be the only traversal defense.** `path.Clean("/foo/../../bar")` returns `/bar`, not an error. By the time the handler joins with the static root, the original intent is gone. The fix is a pre-clean rejection: scan the URL path for any literal `..` segment and 400 it before cleaning. Treat traversal attempts as bad requests, not silent rewrites.
- **ETags must vary by encoding or shared caches poison.** A single ETag for `app.js` (raw) and `app.js.gz` (gzip-encoded) lets a forward proxy hand the gzip body to a gzip-only client. The validator has to encode the encoding choice. Adding `-gzip` / `-br` to the mtime+size tag is one line and closes the cache-poisoning hole.
- **Precompressed siblings need explicit `Content-Type` because `net/http` sniffs from the file head.** The first 512 bytes of `app.js.gz` look like gzip, not JavaScript, so `http.ServeContent` would emit `application/x-gzip` without an override. Deriving `Content-Type` from the *raw* file extension keeps `application/javascript` stable across all three variants — same trick as nginx's `gzip_static`.
- **Don't wire pipeline integration when the integration file is owned by another concurrent agent.** Phase 0hh owns `pipeline.go`. Solution: ship `StaticHandler` as a standalone `http.Handler` users mount themselves (`mux.Handle("/static/", http.StripPrefix(...))`) and document the wiring pattern in godoc. Phase 0hh integrates later by calling the same exported function. Single source of truth, zero merge conflict.
- **Configuration errors surface at first request, not at construction.** `StaticHandler` with `Dir == ""` returns an `http.Handler` that 500s every request rather than panicking or returning `(http.Handler, error)`. The error-returning shape would force every caller to check; the 500-on-request shape lets composition stay one-line and surfaces failure on the first hit (where logs are watched).

### Self-rules

1. **Static asset handlers serve precompressed siblings; they do not compress on demand.** Build step writes `.br`/`.gz`; runtime negotiates and reads bytes. No `compress/gzip` writer in the request path, no brotli library dependency in the runtime module.
2. **Tests that assert `Content-Encoding` or compare compressed bodies must use `&http.Client{Transport: &http.Transport{DisableCompression: true}}`.** Bake into a `rawClient()` helper in any new compression-related test file so it never gets forgotten.
3. **Reject `..` segments before `path.Clean`.** Walk the URL path's `/`-separated tokens and 400 the request if any token equals `..`. `path.Clean` is for safe shape-normalization (collapsing `./`, double slashes); it is not a security boundary.
4. **ETags for negotiated content include the chosen encoding token.** Otherwise shared caches will hand the wrong-encoding body to the wrong client.
5. **Override `Content-Type` from the raw extension when serving compressed siblings.** `http.ServeContent` sniffs the first 512 bytes; for `.gz` / `.br` siblings that gives the wrong type. Set the header before calling `ServeContent`.
6. **When a parallel agent owns the integration site, ship the new capability as a standalone http.Handler with godoc'd mount pattern.** The integrating agent calls the exported function later — no upstream merge conflict, no follow-up debt.
7. **Configuration errors in handler constructors return a 500-on-request handler, not a panic and not a `(handler, error)` pair.** Composition stays one-line; failure surfaces on the first request like any other deploy misconfig.
